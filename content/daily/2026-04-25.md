### 1. Cross-Session Threats in AI Agents: Benchmark, Evaluation, and Algorithms

*arXiv preprint | 2026-04-22T22:40:31Z | Ari Azarafrooz*

> The paper introduces a benchmark and bounded-memory reader for detecting agent attacks whose policy violation only becomes visible when evidence is aggregated across sessions.

The paper targets AI-agent guardrails that judge each message or session in isolation, which leaves a gap for attacks whose harmful payload is distributed across many sessions. It builds CSTM-Bench with executable attack taxonomies, identity anchors, benign confounders, and two 54-scenario splits, then frames detection as selecting enough cross-session evidence for a downstream correlator LLM. The proposed Coreset Memory Reader keeps high-signal fragments under a bounded memory budget, while CSR_prefix measures whether the retained ordering remains stable enough to preserve KV-cache prefix reuse. The abstract reports that session-bound judging and full-log concatenation lose substantial recall on the harder cross-session split, while the bounded-memory reader is the only reader said to retain recall across both shards. The main missing detail is evaluation breadth: the abstract names one correlator family, small shard sizes, no prompt optimization, and does not provide latency, cost, or throughput measurements for the serving-stability claim.

If the claims hold, the work gives agent-serving teams a concrete way to evaluate memory-based security filters while accounting for cache-stable retrieval rather than only detection recall. A key caveat is that the abstract reports only one correlator family and no prompt optimization, so model-specific effects may dominate. Another caveat is that the benchmark scale is small enough that taxonomy coverage and scenario diversity need careful inspection.

### 3. Sparse Forcing: Native Trainable Sparse Attention for Real-time Autoregressive Diffusion Video Generation

*arXiv preprint | 2026-04-23T02:22:25Z | Boxun Xu, Yuming Du, Zichang Liu, Siyu Yang et al.*

> Sparse Forcing adds trainable block-sparse attention and a PBSA GPU kernel to autoregressive video diffusion, targeting lower decoding latency and KV-cache memory without giving up long-horizon visual quality.

Autoregressive video diffusion rollouts face a systems problem similar to long-context generation: attention and KV-cache state grow expensive as horizon length increases. The paper argues that attention repeatedly concentrates on persistent salient visual blocks, then turns that pattern into a trainable sparse mechanism that preserves selected blocks and limits local-window computation to dynamic neighborhoods. It also introduces Persistent Block-Sparse Attention, a GPU kernel meant to make the sparse memory update path practical during training and decoding. The abstract reports VBench improvements over Self-Forcing, 1.11-1.27x decoding speedups depending on generation length, and a 42% lower peak KV-cache footprint. The missing details are the hardware setup, kernel baselines, training cost, and how robust the sparsity policy is outside the reported video settings.

If the implementation is robust, the main engineering value is a concrete sparse-attention path that links model training, KV-cache policy, and GPU execution for lower-latency long video decoding. A key caveat is that the abstract does not disclose hardware or enough kernel-level setup to judge speedup portability. Another caveat is that self-Forcing is the named comparison, so baseline breadth and fairness need checking in the full paper.Text-to-video VBench gains may not fully capture temporal artifacts or failure modes under long rollouts.

### 5. LLM-VeriOpt: Verification-Guided Reinforcement Learning for LLM-Based Compiler Optimization.

*CGO | 2026 | Xiangxin Fang, Jiaqin Kang, Rodrigo Rocha, Sam Ainsworth 0001 et al.*

> LLM-VeriOpt trains a small LLM for LLVM IR peephole optimization by using Alive2 semantic-equivalence feedback inside reinforcement learning.

LLM-based compiler optimization is attractive because it could automate low-level transformations, but unsafe rewrites make raw model outputs hard to use. This paper proposes LLM-VeriOpt, a verification-guided RL method that trains Qwen-3B with Alive2 semantic-equivalence signals as part of a GRPO reward. The target system is an LLVM IR peephole optimizer, with the main claim that verifier feedback improves both correctness and the amount of code the model can safely transform. The abstract reports a 5.4x improvement in successfully modified code over prompted Qwen-3B, 90% verifiably correct output, speedups of 2.3x over O0 code, and performance comparable to LLVM instcombine with some cases outperforming it. The largest missing detail is the evaluation setup: benchmark mix, rewrite distribution, timeout behavior, baseline prompting/fine-tuning fairness, and how the speedups were measured are not clear from the abstract.

If the evaluation holds up, the work is a concrete example of verifier-in-the-loop training for low-level code generation rather than ordinary LLM code synthesis. A key caveat is that the abstract does not reveal benchmark diversity, so the reported gains may depend on narrow peephole patterns. Another caveat is that speedup claims over O0 may be easier to achieve than comparisons against stronger optimization pipelines beyond instcombine-level behavior unless fully contextualized in the paper.

### 6. Q-Hitter: A Better Token Oracle for Efficient LLM Inference via Sparse-Quantized KV Cache.

*MLSys | 2024 | Zhenyu Zhang 0015, Shiwei Liu 0003, Runjin Chen, Bhavya Kailkhura et al.*

> The paper argues that sparse KV-cache retention and low-bit KV quantization interact poorly, then proposes token selection that accounts for both attention importance and quantization friendliness.

Long-context LLM inference pays substantial memory and bandwidth cost to store and load the KV cache, making sparse retention and quantization natural compression levers. The paper's central claim is that Heavy-Hitter-style token retention and low-bit KV-cache quantization are not safely composable: quantization can distort heavy-hitter selection during generation, especially in deeper layers, and high-attention tokens may not be the tokens that quantize best. Q-Hitter changes the token oracle by combining accumulated attention scores with layer-specific quantization-friendliness metrics. The supplied abstract supports the motivation and the stated failure modes, but it does not provide clear quantitative results, model coverage, hardware setup, or baseline details for Q-Hitter. If the full paper validates the claimed interaction, the value is in a more inference-aware KV-cache compression policy rather than another standalone sparsity or quantization recipe.

The likely payoff is a KV-cache compression policy that better matches the joint constraints of sparse attention retention and low-bit inference serving. A key caveat is that the provided abstract is truncated and does not report concrete Q-Hitter performance numbers. Another caveat is that hardware assumptions, kernel/runtime integration, and serving setup are not specified in the supplied text.
