### 1. A Pipelined Collaborative Speculative Decoding Framework for Efficient Edge-Cloud LLM Inference

*arXiv preprint | 2026-03-19T16:51:32Z | Yida Zhang, Zhiyong Gao, Shuaibing Yue, Jie Li et al.*

> PicoSpec is a training-free edge-cloud speculative decoding framework that overlaps edge drafting with cloud verification and compresses verifier-side communication to reduce serving latency.

This paper studies edge-cloud LLM inference, where cloud-only serving is costly and device-only inference is too resource-constrained, but collaboration is often slowed by bandwidth limits and coordination stalls. It proposes PicoSpec, a training-free speculative decoding framework that asynchronously pipelines an edge SLM with a cloud LLM to remove the mutual waiting pattern in vanilla collaborative decoding. It also adds separate rejection sampling with sparse compression so the system avoids repeatedly sending full vocabulary distributions and instead pays a one-time compressed-vocabulary transmission cost. The abstract claims experiments that beat both baselines and prior methods, with up to 2.9x speedup, so the reported gains appear to come from overlapping compute and reducing communication-heavy verification. What the abstract does not make clear is the exact network setting, hardware split, baseline fairness, and whether the method remains strong across different model pairings and acceptance regimes.

If the evaluation is sound, the method offers a practical runtime path for bandwidth-constrained collaborative serving by attacking both pipeline idle time and verifier communication cost. A key caveat is that the reported gains may depend heavily on favorable network conditions, which the abstract does not specify. Another caveat is that baseline fairness is hard to judge because the compared methods and implementation details are not described in the abstract.

### 2. DyMoE: Dynamic Expert Orchestration with Mixed-Precision Quantization for Efficient MoE Inference on Edge

*arXiv preprint | 2026-03-19T17:30:01Z | Yuegui Huang, Zhiyuan Fang, Weiqi Luo, Ruoyu Wu et al.*

> This paper presents a runtime scheme for edge MoE inference that dynamically adjusts expert quantization, protects sensitive layers, and prefetches experts to cut latency under memory and I/O pressure.

MoE models reduce active compute, but on edge devices their many experts still create large memory footprints and repeated I/O stalls, so latency is often dominated by data movement rather than arithmetic. This paper argues that expert importance is skewed and layer sensitivity changes with depth, and uses that observation to drive dynamic mixed-precision quantization, depth-adaptive scheduling, and look-ahead prefetching at runtime. The abstract claims large TTFT and TPOT gains over offloading baselines on commercial edge hardware while preserving accuracy, suggesting the improvement comes from reducing transferred bytes and hiding load stalls instead of changing the MoE architecture itself. The practical value is a systems recipe for making expert-heavy models more usable on constrained devices. The main missing detail is whether the evaluation setup, hardware assumptions, control overhead, and accuracy trade-offs are fair and robust across models and workloads.

If the method generalizes, it offers a practical path to serving MoE models on edge devices by co-designing quantization and data-movement scheduling around the real bottleneck of memory and I/O. A key caveat is that the abstract does not specify the hardware envelope, so the gains may depend strongly on a particular edge platform or storage hierarchy. Another caveat is that dynamic quantization and scheduling may introduce control overhead or implementation complexity that the abstract does not quantify clearly; the abstract is underspecified here for full systems evaluation reproducibility assessment the paper. ### 3. From Servers to Sites: Compositional Power Trace Generation of LLM Inference for Infrastructure Planning

*arXiv preprint | 2026-03-19T01:01:41Z | Grant Wilkins, Fiodar Kazhamiaka, Ram Rajagopal*

> This paper proposes a compositional model that generates multi-scale power traces for LLM inference by separating workload state transitions from configuration-specific power distributions.

This paper targets a gap in datacenter power modeling for LLM serving, where prefill, decode, and idle phases create fast state changes that standard static or nameplate-based models miss. The authors argue that inference power can be modeled compositionally: one component captures workload-driven transitions among operating states, and another captures the power distribution within each state for a given serving configuration. Using measured traces, they synthesize new traces that aggregate from GPU servers up to facility-level load profiles at the temporal granularity needed for infrastructure studies. The abstract reports median absolute energy error below 5% for most tested configurations and claims the generated traces preserve temporal autocorrelation across multiple LLMs, tensor-parallel settings, and GPU generations. The practical value is strong for oversubscription and utility planning, but the abstract does not make the training data requirements, baseline comparisons, or robustness under real production traffic clear.

If the claims hold, this gives infrastructure teams a more realistic way to study LLM serving power at server, rack, row, and facility scales for capacity planning, oversubscription analysis, and grid-facing load characterization. A key caveat is that this is infrastructure modeling, not inference acceleration; its fit is weaker if your main interest is kernel generation or latency-critical serving paths. Another caveat is that the abstract does not specify the baselines, trace collection setup, or how sensitive the method is to deployment-specific calibration data.

### 4. SOL-ExecBench: Speed-of-Light Benchmarking for Real-World GPU Kernels Against Hardware Limits

*arXiv preprint | 2026-03-19T17:30:02Z | Edward Lin, Sahil Modi, Siva Kumar Sastry Hari, Qijing Huang et al.*

> A Blackwell-oriented CUDA-kernel benchmark that measures how much an optimizer closes the gap to an analytical hardware Speed-of-Light bound instead of just beating a software baseline.

This paper argues that GPU-kernel benchmarks for agentic optimizers should be judged by distance to hardware-efficient execution, not just by speedup over a software baseline. It introduces SOL-ExecBench, a Blackwell-focused suite of 235 CUDA kernel optimization problems drawn from 124 real AI models across forward and backward workloads, precision modes, and modalities. The core idea is to pair each task with an analytically derived Speed-of-Light bound from the SOLAR pipeline and score candidate kernels by how much of the baseline-to-SOL gap they close. The abstract also describes a hardened evaluation harness with clock locking, L2 cache clearing, isolated subprocesses, and static checks, which supports more controlled evaluation of agentic optimizers. The likely value is a more stable, hardware-aware target for kernel-generation research, but the abstract does not say how tight the analytical bounds are or how fair the release-defined baseline is.

If the bound model is credible, this benchmark could make CUDA-kernel optimization work easier to compare across real AI workloads and better aligned with actual hardware limits. A key caveat is that the benchmark is tightly tied to NVIDIA Blackwell, so conclusions may not transfer to older GPUs or other hardware backends. ### 5. SpecForge: A Flexible and Efficient Open-Source Training Framework for Speculative Decoding

*arXiv preprint | 2026-03-19T07:28:56Z | Shenggui Li, Chao Wang, Yikai Zhu, Yubo Wang et al.*

> An open-source EAGLE-3 training stack plus released draft models aimed at removing two practical blockers to speculative decoding: scalable draft training and the lack of usable drafts.

Speculative decoding is a promising way to cut autoregressive LLM latency, but practical deployment is often limited by weak draft models and by the cost of training them at scale. This paper presents SpecForge, an open-source framework for speculative decoding training with EAGLE-3 support, combining target-draft decoupling, hybrid parallelism, optimized training kernels, and integration with production inference engines. It also introduces SpecBundle, a set of production-oriented EAGLE-3 draft models for mainstream open-source LLMs, so the work covers both infrastructure and draft availability. The abstract claims up to 9.9x faster EAGLE-3 training for Qwen3-235B-A22B and up to 4.48x end-to-end inference speedup on SGLang, which makes it directly relevant to serving efficiency. The biggest missing details are the hardware setup, baseline fairness, and the quality or acceptance-rate tradeoffs behind those speedups.

If these claims hold under fair baselines, the work could reduce the cost of building deployable draft models and make speculative decoding easier to operationalize in production serving stacks. A key caveat is that the abstract does not specify the hardware, baseline quality, or cost model behind the reported training gains. Another caveat is that it is unclear how the claimed speedups interact with output quality, acceptance rate, or other serving-side tradeoffs that matter in practice for speculative decoding evaluation, and the abstract is underspecified on those points exactly tied to EAGLE-3, Qwen3-235B-A22B, and SGLang, so the extent of generalization to other models, runtimes, or speculative schemes is not clear.

### 6. Fast Inference from Transformers via Speculative Decoding

*International Conference on Machine Learning | 2022-11-30 | Yaniv Leviathan, Matan Kalman, Yossi Matias*

> A draft-and-verify decoding scheme for Transformers that aims to speed autoregressive generation while preserving the target model's exact output distribution.

This paper targets the sequential decoding bottleneck in autoregressive Transformer inference and introduces speculative decoding as a systems-level acceleration method. The core idea is to let a cheaper draft process propose tokens and have the target model verify them exactly, so the final distribution is claimed to remain unchanged. From the abstract, the strongest supported point is the formalization of this draft-plus-verification procedure and its exactness claim, not a detailed empirical speedup story. If the method works as described, it offers a general inference pattern for reducing expensive target-model generation work, but the abstract does not make the evaluation setup, hardware assumptions, verification overhead, or practical gain regime clear.

For inference systems work, the main value is a reusable exact draft-and-verify pattern that could translate model asymmetry into lower decoding cost if verification is cheap enough. A key caveat is that the abstract is underspecified on empirical validation, so practical latency or throughput gains cannot be judged from it alone. Another caveat is that verification cost may erase the benefit, but the abstract does not explain the operating regime where this tradeoff is favorable enough to matter in deployment.
