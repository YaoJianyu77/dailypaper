### 1. FluxMoE: Decoupling Expert Residency for High-Performance MoE Serving

*arXiv preprint | 2026-04-03T04:16:01Z | Qingxiu Liu, Cyril Y. He, Hanser Jiang, Zion Wang et al.*

> This paper presents an MoE serving design that stops keeping expert weights permanently resident on GPU and instead pages experts in on demand to free memory for KV cache and other throughput-critical state.

The paper targets a core MoE serving bottleneck: expert weights occupy GPU memory even though only a small subset is active per token, while KV cache capacity is the resource that more directly drives throughput. FluxMoE addresses this by decoupling expert residency from persistent GPU memory, treating experts as transient paged resources that are materialized when needed and evicted immediately after use. Implemented on top of vLLM, the system is positioned as a serving runtime change rather than a model-side modification. The abstract claims up to 3.0x throughput improvement over vLLM in memory-intensive settings without hurting model fidelity, which makes the method relevant for constrained-memory MoE deployment. The biggest missing detail is the actual cost model and operating envelope: the abstract does not make clear the paging overheads, hardware assumptions, routing locality, or when streaming experts stops paying off.

If the tradeoff holds in practice, this could improve MoE serving efficiency under tight GPU memory budgets by converting unused expert residency into higher effective KV cache capacity and better throughput. A key caveat is that the abstract does not specify the paging source, bandwidth bottlenecks, or overlap strategy, so the gains may depend heavily on hardware and workload locality. Another caveat is that memory-intensive regimes' is broad; the reported improvement may not generalize to less constrained deployments or different routing patterns.

### 2. AXELRAM: Quantize Once, Never Dequantize

*arXiv preprint | 2026-04-03T02:03:38Z | Yasushi Nishida*

> This paper proposes an SRAM-oriented attention design that operates on quantized KV-cache indices directly, aiming to remove dequantization and most query-side multiply cost.

This paper targets KV-cache handling in LLM inference and argues that attention scores can be computed directly from quantized cache indices instead of reconstructing full-precision values. The core idea is a fixed codebook enabled by an orthogonal-transform-based quantization scheme, plus an asymmetric hardware path that applies the transform on write and uses table lookup on read without inverse transform. The abstract claims a mathematical 102.4x reduction in per-query multiplications and reports multi-seed evaluation showing an important failure mode: some models exhibit catastrophic perplexity spikes from sign-pattern sensitivity, while others remain stable. The paper then proposes a gradient-free sign-pattern selection procedure to remove those spikes at no added hardware cost. What still needs a full read is whether end-to-end latency, SRAM area and energy tradeoffs, and baseline fairness actually support the claimed systems benefit beyond the arithmetic-count reduction.

If the hardware and accuracy tradeoffs hold, the method could reduce KV-cache read-side compute and simplify attention-serving datapaths for memory-constrained LLM inference. A key caveat is that the main systems gain stated in the abstract is an arithmetic-count reduction, not a clearly demonstrated end-to-end serving improvement. Another caveat is that the method appears sensitive to model-dependent sign patterns, which raises portability and reproducibility concerns across architectures and scales.

### 3. TokenDance: Scaling Multi-Agent LLM Serving via Collective KV Cache Sharing

*14 pages, 14 figures, arXiv:submit/7438760 [cs.DC], preprint under review | 2026-04-03T16:04:40Z | Zhuohang Bian, Feiyang Wu, Chengrui Zhang, Hangcheng Dong et al.*

> This paper proposes a multi-agent LLM serving system that exploits round-level all-gather structure to share KV cache collectively instead of duplicating shared prompt blocks across agents.

This paper targets a specific inefficiency in synchronized multi-agent LLM serving: agents repeatedly receive the same shared context, but their KV caches are still stored and reused largely per request. The proposed system, TokenDance, introduces a collective KV reuse step across a full round and stores related agent caches as block-sparse diffs against a master copy, so both reuse cost and storage growth scale better with agent count. The abstract claims this reduces KV redundancy substantially, improves prefill performance, and increases the number of concurrent agents that can be served under an SLO compared with vLLM prefix caching and position-independent caching. For systems readers, the main value is the explicit co-design between multi-agent communication structure and KV-cache management rather than another generic cache heuristic. The biggest missing detail is how robust the method is under less synchronized workloads, what the exact hardware and SLO setup are, and whether the reported baselines are fully comparable.

If the method generalizes, it offers a practical way to push multi-agent LLM serving capacity by converting communication-level redundancy into lower memory footprint and lower prefill overhead. A key caveat is that the gains may depend heavily on synchronized round structure and may weaken for asynchronous or highly divergent agent workloads. Another caveat is that the abstract does not clarify implementation overheads, communication costs, or failure modes of the collective reuse mechanism at scale.

### 4. Fast NF4 Dequantization Kernels for Large Language Model Inference

*7 pages, 4 figures, EMC2 Workshop at ASPLOS 2026 | 2026-04-02T22:15:57Z | Xiangbo Qi, Chaoyi Jiang, Murali Annavaram*

> This paper targets NF4 dequantization as an LLM inference bottleneck on NVIDIA GPUs and proposes a small shared-memory optimization that speeds the kernel up while preserving compatibility with existing HuggingFace-style workflows.

This paper studies a practical systems bottleneck in NF4-quantized LLM inference: weights save memory, but runtime dequantization back to FP16 can dominate execution on current NVIDIA GPUs. The authors propose a lightweight kernel optimization that exploits shared memory and simpler indexing logic, with the stated goal of reducing memory access cost and instruction count without breaking existing software stacks. The abstract claims 2.0-2.2x kernel speedup against BitsAndBytes across Gemma 27B, Qwen3 32B, and Llama3.3 70B, plus up to 1.54x end-to-end improvement on A100-class hardware. If those results hold, the value is immediate for quantized serving on installed GPU fleets, but the abstract leaves important details unclear around exact evaluation setup, baseline fairness, and how broadly the method transfers beyond the reported kernels and hardware.

If robust, this is a low-complexity serving optimization for NF4-based LLM deployment that could improve throughput or latency on existing A100-class systems without changing model formats. A key caveat is that the abstract centers on A100-era hardware, so portability to other architectures is not yet clear. Another caveat is that the reported end-to-end gains depend heavily on workload mix and system configuration, which the abstract does not fully specify.

### 5. SkillRT: Compiling Skills for Efficient Execution Everywhere

*arXiv preprint | 2026-04-03T15:11:45Z | Le Chen, Erhu Feng, Yubin Xia, Haibo Chen*

> This paper frames reusable agent skills as compilable programs and proposes a compiler-runtime stack that adapts them to different model and harness capabilities for better portability and lower execution cost.

The paper argues that agent skills are currently passed around as raw prompt context, which makes them fragile across different LLMs and agent harnesses and wastes execution budget. It proposes SkillRT, a compiler and runtime that treats skills like code, profiles primitive capabilities of each model-harness pair, and then performs capability-based compilation, environment binding, concurrency extraction, JIT code solidification, and adaptive recompilation. The abstract claims improved task completion across eight LLMs and three harnesses, along with lower token use, higher parallelism, and large latency reductions from code solidification. For systems readers, the interesting angle is the compiler/runtime co-design for cross-model agent execution rather than standard prompting. The biggest missing detail is what the capability model actually looks like, how fair the baselines are, and how much of the gain comes from execution caching or task-specific restructuring versus genuinely better portability.

If the method holds up, it suggests a practical path to serving reusable agent skills with lower latency and token cost while reducing environment-specific breakage. A key caveat is that the abstract leaves unclear how general the capability abstraction is beyond the evaluated benchmarks and harnesses. Another caveat is that large latency improvements may depend heavily on solidification assumptions that do not transfer to stateful or open-ended agent tasks.
