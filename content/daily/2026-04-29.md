### 1. AHASD: Asynchronous Heterogeneous Architecture for LLM Adaptive Drafting Speculative Decoding on Mobile Devices

*arXiv preprint | 2026-04-28T07:42:41Z | Ma zirui, Fan Zhihua, Li Wenxing, Wu Haibin et al.*

> AHASD proposes a mobile NPU-PIM speculative decoding architecture that overlaps draft generation and target-model verification while using confidence- and timing-aware controls to reduce wasted drafting work.

Speculative decoding can improve LLM inference, but mobile single-NPU-PIM execution creates a scheduling problem: synchronous operator-level execution leaves hardware idle, while naive asynchronous drafting can waste compute when draft length fluctuates. AHASD moves to task-level DLM-TLM decoupling, running drafting on LPDDR5-PIM and verification on a single NPU, with entropy-history-aware drafting control and time-aware pre-verification control to limit low-confidence invalid drafts. The design also adds attention algorithm units and gated task scheduling units in PIM to localize attention work and enable sub-microsecond task switching. The abstract reports experiments across different LLMs and adaptive drafting algorithms, claiming up to 4.2x throughput and 5.6x energy-efficiency gains over a GPU-only baseline, plus 1.5x throughput and 1.24x energy-efficiency gains over a GPU+PIM baseline, with under 3% DRAM-area overhead. The main missing detail is the evaluation substrate and fairness of the baselines, including whether the results are measured, simulated, or modeled.

If the evaluation holds, the useful contribution is a concrete hardware-runtime co-design path for making speculative decoding practical on constrained mobile LLM inference platforms. A key caveat is that the abstract does not disclose enough evaluation detail to judge baseline fairness or reproducibility. Another caveat is that the specialized LPDDR5-PIM units may limit applicability outside the assumed mobile NPU-PIM architecture constraints.

### 2. Pythia: Toward Predictability-Driven Agent-Native LLM Serving

*arXiv preprint | 2026-04-28T17:41:53Z | Shan Yu, Junyi Shu, Yuanjiang Ni, Kun Qian et al.*

> Pythia treats multi-agent workflow structure as a serving-layer signal for making LLM inference traffic more predictable and improving throughput and job completion time.

Multi-agent LLM applications introduce workflow topology that can constrain agent behavior and make request patterns less arbitrary than generic chat traffic. The paper argues that current LLM serving systems ignore this structure, leading to low prefix cache hit rates, long-context resource contention, and scaling-related queuing delays. Pythia exposes workflow semantics to the serving layer through a simple interface so the runtime can exploit predictability in agent-native workloads. The abstract claims substantial throughput and job completion-time gains over state-of-the-art baselines, supported by analysis of production traces from an agent-serving platform and an internal coding assistant. The main missing details are the exact interface, the concrete scheduling/cache/scaling mechanisms, hardware assumptions, and baseline setup.

If the mechanism is sound, the payoff is a serving design that turns agent workflow structure into practical efficiency gains for long-context, multi-step LLM applications. A key caveat is that the abstract does not describe the actual algorithms, so the source of the performance gain is unclear. Another caveat is that the evaluation details are missing, making it impossible to judge hardware assumptions, workload realism, or baseline fairness from the abstract alone.

### 3. SpecFed: Accelerating Federated LLM Inference with Speculative Decoding and Compressed Transmission

*IEEE International Symposium on Information Theory (ISIT), 2026 | 2026-04-28T15:44:50Z | Ce Zheng, Xinghan Wang, Jiahong Ning, Yuxuan Shi et al.*

> SpecFed targets federated LLM inference latency by combining speculative decoding with top-K probability transmission and server-side reconstruction to reduce per-token communication.

Federated LLM inference can improve edge-side generation by aggregating distributed model predictions, but autoregressive decoding forces repeated full-model forward passes and frequent communication between workers and a server. The paper frames the main bottleneck as transmitting full token probability distributions for each draft token, which can dominate end-to-end latency. SpecFed introduces speculative decoding for parallel draft processing and compresses worker communication by sending top-K probability information, then reconstructing distributions on the server with two proposed strategies. The abstract claims theoretical bounds for reconstruction error, aggregation bias, and acceptance-rate bias, plus experiments showing high generation fidelity and lower communication overhead. The missing details are the actual speedups, network assumptions, model sizes, baseline choices, and whether communication savings translate into wall-clock throughput gains.

If the claimed tradeoff holds, the method could make bandwidth-limited federated or edge LLM serving more practical without fully abandoning aggregated predictions. A key caveat is that the abstract gives no numerical speedup, throughput, latency, or bandwidth figures. Another caveat is that evaluation setup, model scale, hardware, and network assumptions are unspecified in the provided text.

### 4. Feasible-First Exploration for Constrained ML Deployment Optimization in Crash-Prone Hierarchical Search Spaces

*arXiv preprint | 2026-04-27T23:58:24Z | Christian Lysenstøen*

> The paper frames constrained ML deployment tuning as a crash-prone hierarchical search problem and proposes feasible-first exploration before TPE exploitation.

Production deployment tuning often couples model choice, quantization, runtime backend, and serving configuration, which creates many invalid or constraint-violating configurations. This paper targets the small-budget regime where standard black-box optimizers can waste trials on crashes, memory failures, or latency violations. Its Thermal Budget Annealing method first maps feasible regions, using early trial timeouts and temporary categorical-subspace blacklisting, then warm-starts TPE for exploitation. The abstract reports synthetic benchmarks and real GPU deployment experiments on five pre-trained vision models across H100, A100, RTX 5080, L4, and T4, claiming better model-family discovery and less wasted budget than cold-start TPE. The main missing detail is the magnitude and robustness of the gains across constraints, hardware targets, and optimizer baselines.

If the claims hold, the method is useful for inference deployment search where avoiding failed trials can matter as much as finding a slightly better valid configuration. A key caveat is that the abstract does not report numerical gains, variance, or failure rates, so baseline fairness needs close checking. Another caveat is that the real deployment evidence is on pre-trained vision models, not explicitly on LLM serving workloads or CUDA kernel generation workflows.

### 5. QFlash: Bridging Quantization and Memory Efficiency in Vision Transformer Attention

*arXiv preprint | 2026-04-28T07:13:26Z | Sehyeon Oh, Yongin Kwon, Jemin Lee*

> QFlash proposes an integer-only FlashAttention path for vision transformer attention, moving softmax into the integer domain inside a single Triton kernel.

FlashAttention reduces attention memory traffic through tiling, but its online softmax still depends on floating-point arithmetic for numerical stability, which blocks a fully quantized path. QFlash targets three concrete barriers: scale growth during tile accumulation, inefficient GPU shift-based exponentials, and scale-granularity constraints for integer comparison. The method implements end-to-end integer softmax and attention as a single Triton kernel. The abstract reports evaluation on seven ViT, DeiT, and Swin attention workloads, with up to 6.73x speedup over I-ViT, up to 8.69x speedup on Swin, 18.8% lower energy than FP16 FlashAttention, and no Top-1 loss on ViT/DeiT. The main missing detail is whether the kernel design and quantization constraints transfer cleanly to LLM attention shapes, long contexts, and production serving baselines.

If the claims hold under fair kernel baselines, the work could be useful for quantized attention implementations where memory efficiency and integer execution matter more than general model-level changes. A key caveat is that evaluation scope in the abstract is limited to ViT, DeiT, and Swin attention workloads, not LLM serving workloads. Another caveat is that the abstract omits key setup details needed to judge baseline fairness and hardware dependence.

### 6. CUDA Kernel Optimization and Counter-Free Performance Analysis for Depthwise Convolution in Cloud Environments

*arXiv preprint | 2026-04-28T09:29:53Z | Huriyeh Babak, Melanie Schaller*

> A controlled CUDA depthwise-convolution study shows that memory locality, on-chip reuse, and warp tiling can yield meaningful speedups even when cloud profiling counters are unavailable.

This paper studies depthwise convolution for S4ConvD as an operator-level GPU optimization problem, with the model, dataset, and training setup fixed while only CUDA kernel implementations change. It compares naive, global-memory-coalesced, shared-memory cache-blocked, and warp-tiled variants across forward, input-gradient, and weight-gradient paths. The claimed gains come from better memory access, locality, and on-chip data reuse, with the warp-tiled kernel reducing convolution runtime by 3.26x versus the naive CUDA baseline and improving end-to-end training speed by 1.29x. The analysis uses CUDA-event timing, execution-path decomposition, analytic memory-traffic modeling, effective bandwidth, and roofline reasoning instead of hardware counters. The abstract is specific about the optimization path but underspecified on GPU type, cloud setup, tensor shapes, and baseline implementation details beyond noting that PyTorch is not used as a controlled architectural baseline.

If the methodology holds under scrutiny, it gives CUDA engineers a practical way to reason about kernel bottlenecks in cloud settings where Nsight-style counter access is limited. A key caveat is that the abstract does not specify hardware, tensor shapes, or cloud environment, which limits reproducibility assessment. Another caveat is that pyTorch is used only for validation and runtime context, so framework-level baseline fairness needs checking in the full paper.
