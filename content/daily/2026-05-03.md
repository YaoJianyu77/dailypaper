### 1. VitaLLM: A Versatile, Ultra-Compact Ternary LLM Accelerator with Dependency-Aware Scheduling

*arXiv preprint | 2026-04-30T04:07:21Z | Zi-Wei Lin, Tian-Sheuan Chang*

> VitaLLM is a hardware-software co-designed ternary LLM accelerator for edge inference that targets bandwidth-bound decoding, workload imbalance, and dependency stalls with specialized ternary compute, mixed-precision attention support, KV-fetch pruning, and scheduling.

Ternary LLMs can reduce model footprint, but the abstract frames edge deployment as still limited by memory bandwidth, power, workload imbalance, and decode-stage dependencies. VitaLLM addresses this with a dual-core architecture: TINT-Cores handle large ternary projections, while a BoothFlex-Core supports mixed-precision attention across prefill and decode. It also adds Leading One Prediction to avoid redundant KV-cache fetches and dependency-aware scheduling to overlap or hide nonlinear-operation latency. The paper reports a TSMC 16nm implementation with 70.70 tokens/s decoding throughput, 0.223 mm^2 area, 65.97 mW power, and 17.4 TOPS/mm^2/W FOM, claiming superiority over prior accelerators. The abstract does not make the workload, model size, sequence length, baseline set, or measurement methodology clear, so the claimed efficiency needs a careful full-paper check.

If the reported design holds under realistic LLM workloads, it offers a compact edge-inference template for ternary models where bandwidth and power dominate system cost. A key caveat is that the abstract does not expose enough evaluation detail to judge workload realism or baseline fairness. Another caveat is that the accelerator is specialized for ternary inference, so applicability to common mixed-precision or non-ternary LLM deployments may be limited unless BoothFlex-BS is convincing in the full paper/web appendix and evidence claims are checked against the paper text, rather than assumed from the abstract alone; no code or reproducibility path is described in the provided input.

### 2. TwinGate: Stateful Defense against Decompositional Jailbreaks in Untraceable Traffic via Asymmetric Contrastive Learning

*arXiv preprint | 2026-04-30T13:44:01Z | Bowen Sun, Chaozhuo Li, Yaodong Yang, Yiwei Wang et al.*

> TwinGate targets decompositional jailbreaks in anonymized LLM traffic with a stateful dual-encoder monitor designed to run alongside prefill with low serving overhead.

This paper addresses a safety problem that becomes harder in real LLM serving settings: malicious intent can be split across benign-looking, anonymized, interleaved requests. TwinGate uses a stateful dual-encoder design, with asymmetric contrastive learning to bring intent-matched malicious fragments together and a frozen parallel encoder to reduce false positives from benign topical overlap. The abstract claims each request needs only one lightweight forward pass that can run in parallel with the target model's prefill phase, so the system is positioned as a low-latency serving defense rather than a heavyweight generative monitor. Evidence is reported on a 3.62M-instruction dataset covering 8,600 malicious intents, with high recall, low false positives, adaptive robustness, and better throughput and latency than baselines. The main missing details are the exact metrics, hardware setup, baseline implementations, and how the state mechanism works under fully anonymized interleaving.

If the latency claim holds, this is useful for LLM serving teams that need cross-request safety state without putting a generative model in the critical path. A key caveat is that the abstract gives no concrete latency, throughput, recall, or false-positive numbers, so the strength of the systems claim is hard to judge. Another caveat is that the mechanism for state under anonymized interleaving is central but underspecified in the abstract exposed here.

### 3. CuLifter: Lifting GPU Binaries to Typed IR

*arXiv preprint | 2026-04-30T06:34:54Z | Jisheng Zhao, Huanzhi Pu, Shinnung Jeong, Chihyo Ahn et al.*

> CuLifter turns SASS GPU binaries into typed LLVM IR by making register type recovery the core lifting problem.

CuLifter addresses a practical blocker in GPU binary analysis: SASS binaries lose source-level type distinctions because GPU compilers map values into a unified register file. The paper frames type recovery from that untyped register file as the central lifting problem and builds a SASS-to-LLVM IR pipeline using constraint propagation with conflict detection, explicit control-flow reconstruction, and multi-instruction pattern aggregation. The abstract reports evaluation across eight benchmark suites, 24,437 GPU functions, and 919 cubins from open-source applications, vendor libraries, and optimized ML runtimes, with 99.98% of functions lifted to valid LLVM IR. It also reports an ablation where disabling type recovery drops the x86 pass rate from 73.8% to 0%, supporting the claim that type recovery is critical for semantic correctness. The biggest missing detail is how semantic equivalence, baseline fairness, GPU architecture coverage, and failure cases are defined beyond the headline lift and pass rates.

If the method generalizes, it could make SASS-level inspection, retargeting, optimization analysis, and ML runtime reverse engineering more practical. A key caveat is that the abstract does not specify the semantic equivalence oracle in enough detail to judge correctness rigorously. Another caveat is that the reported lift rate may hide important unsupported instruction patterns, architectures, or heavily optimized kernels in the small failure set without a full read.

### 4. RCW-CIM: A Digital CIM-based LLM Accelerator with Read-Compute/Write

*arXiv preprint | 2026-04-30T03:52:24Z | Yan-Cheng Guo, Tian-Sheuan Chang, Jian-Wei Su*

> The paper proposes a fabricated digital CIM accelerator that targets LLM prefill and decode latency by reducing CIM weight-update overhead, nonlinear-operator dependency latency, and memory movement.

Digital CIM is positioned as a way to reduce DRAM traffic for LLM inference while preserving precision, but the abstract argues that weight-update latency becomes a bottleneck when model weights exceed a single CIM macro. RCW-CIM addresses this with a read-compute/write architecture, nonlinear operator fusion based on partial accumulation and group-based approximation, and a WS-OCS dataflow for the prefill phase. The abstract reports latency and memory-access reductions on Llama2 workloads, including 21.59% lower decoding compute latency, 69.17% latency reduction from nonlinear operator fusion, and 49.76% overall prefill latency reduction for 1024 tokens. It also states the chip was fabricated in TSMC 22 nm CMOS, runs at 100 MHz, and reaches 3.28 TOPS and 42.3 TOPS/W with dual DDR5-6400 memory for an INT4-weight Llama2 setup. The main missing detail is how baselines are defined and whether the reported gains hold across model sizes, sequence lengths, operators, and accuracy constraints introduced by approximation.

If the evaluation is fair, the work is useful for hardware and runtime designers studying where CIM-based LLM inference gains come from beyond raw matrix multiplication throughput. A key caveat is that baseline fairness is underspecified in the abstract, so the reported percentage reductions may be difficult to interpret without the full paper. Another caveat is that the approximation used in nonlinear operator fusion may affect accuracy, but the abstract does not state the quality impact or validation method.

### 5. Strait: Perceiving Priority and Interference in ML Inference Serving

*arXiv preprint | 2026-04-30T17:55:28Z | Haidong Zhao, Nikolaos Georgantas*

> Strait is a GPU inference serving system that predicts transfer contention and kernel interference to schedule dual-priority traffic with fewer high-priority deadline misses.

Strait targets GPU inference serving under high utilization, where on-prem deployments may need priority differentiation and tighter deadline control. The paper argues that existing serving support is limited by weak task prioritization and latency estimates that do not adequately handle concurrent execution. Its mechanism combines contention-aware latency prediction for data transfer, adaptive modeling of kernel execution interference, and priority-aware scheduling for dual-priority request streams. The abstract reports evaluations under intense workloads, with high-priority deadline violations reduced by 1.02 to 11.18 percentage points, acceptable low-priority costs, and more equitable behavior than software-defined preemption approaches. The largest missing detail is the evaluation setup: model types, GPU configuration, workload mix, and baseline tuning are not clear from the abstract.

If the evaluation holds up, the main engineering value is a serving-level way to improve priority isolation and deadline satisfaction under high GPU utilization without requiring low-level preemption. A key caveat is that the abstract does not establish whether the workloads include LLM inference or only general DNN serving. Another caveat is that hardware configuration, GPU count, model mix, and request distribution are unspecified, limiting reproducibility judgment from the abstract alone.

### 6. DeepSpeed Inference: Enabling Efficient Inference of Transformer Models at Unprecedented Scale

*SC22: International Conference for High Performance Computing, Networking, Storage and Analysis | 2022-06-30 | Reza Yazdani Aminabadi, Samyam Rajbhandari, Minjia Zhang, Ammar Ahmad Awan et al.*

> DeepSpeed Inference is presented as a transformer serving system that combines multi-GPU execution, optimized kernels, and CPU/GPU/NVMe memory support for latency- and throughput-constrained inference.

Transformer inference at large scale is constrained by latency, throughput, and memory capacity, especially when models exceed a single GPU's practical serving envelope. This paper presents DeepSpeed Inference as a serving system that combines multi-GPU execution, transformer kernel optimizations, and heterogeneous CPU/GPU/NVMe memory support. The claimed mechanism is system-level coordination across parallel execution, optimized kernels, and an offload-capable memory hierarchy rather than a model-side change. The supplied abstract supports only the existence of these components and the goal of scaling serving under latency and throughput constraints; it does not report concrete measurements, baselines, hardware details, or workload sizes. Its practical value is in framing inference efficiency as runtime, kernel, and memory-management co-design, but a full read must verify where the actual gains come from and how much offload affects latency.

If the claims hold, the work is useful for understanding how inference serving efficiency can come from coordinated runtime, kernel, and memory-system design. A key caveat is that the provided abstract is too high-level to judge baseline fairness or actual performance impact. Another caveat is that offloading through CPU or NVMe may improve capacity while hurting latency, but the abstract does not quantify that tradeoff.
