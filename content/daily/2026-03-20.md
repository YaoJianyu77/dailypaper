### 1. A Pipelined Collaborative Speculative Decoding Framework for Efficient Edge-Cloud LLM Inference

*arXiv preprint | 2026-03-19T16:51:32Z | Yida Zhang, Zhiyong Gao, Shuaibing Yue, Jie Li et al.*

> This paper presents a training-free edge-cloud speculative decoding framework that overlaps edge-side drafting with cloud-side verification and reduces verifier communication with compressed rejection sampling.

This paper targets edge-cloud LLM serving, where pure cloud inference is costly and pure on-device inference is resource-constrained. It argues that standard speculative decoding is a poor fit for this setting because edge and cloud stages can block each other and because sending full vocabulary distributions adds substantial network latency. PicoSpec addresses this with an asynchronous pipeline that runs an edge SLM and a cloud LLM concurrently, plus separate rejection sampling with sparse compression so rejection can be completed after a one-time transfer of compressed vocabulary information. The abstract claims the method outperforms baselines and prior edge-cloud approaches, with up to 2.9x speedup. The main missing detail is that the abstract does not specify the model pairings, hardware setup, network conditions, or baseline fairness, so it is still unclear where the gain comes from in practice and how reproducible it is.

If the claims hold, the framework could make edge-cloud LLM serving more practical by improving latency without retraining and by reducing the network cost of speculative verification. A key caveat is that the abstract does not reveal whether the reported speedup holds under realistic wide-area network conditions or only in favorable settings. Another caveat is that the communication savings may depend heavily on vocabulary sparsity and could weaken if token distributions are less compressible than expected.

### 2. DyMoE: Dynamic Expert Orchestration with Mixed-Precision Quantization for Efficient MoE Inference on Edge

*arXiv preprint | 2026-03-19T17:30:01Z | Yuegui Huang, Zhiyuan Fang, Weiqi Luo, Ruoyu Wu et al.*

> A runtime framework for edge MoE inference that dynamically changes expert precision, schedules layers by sensitivity, and prefetches ahead to reduce memory and I/O-driven latency.

MoE models reduce compute per token, but on edge devices their many experts still create large memory footprints and off-chip I/O stalls that can dominate latency. This paper argues that expert importance is skewed and depth-dependent, and uses that observation to replace static quantization with a dynamic mixed-precision runtime policy. DyMoE combines importance-aware expert quantization, depth-adaptive scheduling for sensitive layers, and look-ahead prefetching to overlap data movement with compute. The abstract claims experiments on commercial edge hardware with 3.44x-22.7x lower TTFT and up to 14.58x faster TPOT than state-of-the-art offloading baselines while preserving accuracy. The main missing details are the exact hardware and model setup, the runtime overhead and decision policy, and how fair the baseline and accuracy comparisons are.

If the method is robust, it offers a practical path to making MoE models usable on edge systems by trading precision and data movement more selectively instead of relying on static policies. A key caveat is that the abstract does not specify hardware, model sizes, or baseline configuration, so the reported gains may depend heavily on a narrow setup. Another caveat is that runtime orchestration may introduce control overhead or implementation complexity that is not visible from the abstract alone.

### 3. From Servers to Sites: Compositional Power Trace Generation of LLM Inference for Infrastructure Planning

*arXiv preprint | 2026-03-19T01:01:41Z | Grant Wilkins, Fiodar Kazhamiaka, Ram Rajagopal*

> A compositional trace generator models LLM inference power by separating workload state transitions from within-state power distributions, then aggregates learned traces from GPU servers to facility load profiles.

This paper targets a gap between traditional datacenter power models and the bursty behavior of LLM inference, where GPUs rapidly alternate among prefill, decode, and idle phases and large deployments can synchronize those shifts into meaningful facility-level load swings. It proposes a compositional trace-generation framework that learns workload-driven operating-state transitions and configuration-specific power distributions from measured traces, then synthesizes traces for new traffic conditions and serving configurations. The generated traces roll up from GPU servers to rack, row, and site scales, making the work relevant to provisioning, oversubscription studies, power modulation, and utility planning rather than only server-local analysis. The abstract claims median absolute energy error below 5% for most tested configurations and says temporal autocorrelation is preserved across multiple LLMs, tensor-parallel settings, and GPU generations. The biggest missing detail is how the measurements were collected and how well the model generalizes to unseen serving stacks, workload mixes, and non-GPU facility components.

If the generalization holds, the method provides a more realistic input for datacenter and utility studies than flat nameplate estimates or static trace replay, especially for capacity and control analysis of AI sites. A key caveat is that the abstract does not specify the measurement setup, so reproducibility and data-collection burden are unclear. Another caveat is that generalization may depend strongly on serving software, batching policy, and accelerator details that are not described in the abstract.

### 4. SOL-ExecBench: Speed-of-Light Benchmarking for Real-World GPU Kernels Against Hardware Limits

*arXiv preprint | 2026-03-19T17:30:02Z | Edward Lin, Sahil Modi, Siva Kumar Sastry Hari, Qijing Huang et al.*

> This paper introduces a Blackwell-targeted CUDA kernel benchmark that scores optimizers by how much they close the gap to analytically derived hardware speed-of-light bounds, rather than only by beating software baselines.

This paper argues that kernel-generation and kernel-optimization systems are now limited by weak evaluation targets: most benchmarks reward relative speedup over software baselines instead of measuring closeness to hardware-efficient execution. It proposes SOL-ExecBench, a benchmark of 235 CUDA kernel optimization problems drawn from 124 AI models, paired with SOLAR, a pipeline that derives hardware-grounded speed-of-light bounds and a SOL Score that measures how much of the remaining gap a candidate kernel closes. The benchmark is positioned around NVIDIA Blackwell GPUs and includes forward and backward workloads across BF16, FP8, and NVFP4, plus a sandboxed harness meant to reduce reward hacking with clock locking, cache clearing, isolated execution, and static checks. The practical value is a more stable target for evaluating agentic kernel optimizers and low-level code generators against hardware limits rather than against mutable software implementations. The biggest missing detail from the abstract is whether the analytical bounds are tight and useful in practice, and how sensitive the benchmark is to baseline choice, workload mix, and Blackwell-specific assumptions.

If the bounds and scoring are well calibrated, this could give kernel-generation and optimization systems a more meaningful target tied to hardware limits, especially for modern mixed-precision AI workloads. A key caveat is that the abstract does not show whether the analytical speed-of-light bounds are accurate enough to serve as trustworthy targets. Another caveat is that the benchmark is tightly tied to NVIDIA Blackwell GPUs, so portability to other architectures is unclear from the abstract alone.

### 5. SpecForge: A Flexible and Efficient Open-Source Training Framework for Speculative Decoding

*arXiv preprint | 2026-03-19T07:28:56Z | Shenggui Li, Chao Wang, Yikai Zhu, Yubo Wang et al.*

> An open-source systems stack for training EAGLE-3 draft models at scale and turning them into deployable speculative decoding speedups.

Speculative decoding is one of the main ways to reduce autoregressive LLM latency, but deployment is often blocked by weak draft models and the lack of scalable training infrastructure. SpecForge addresses that gap with an open-source EAGLE-3 training framework built around target-draft decoupling, hybrid parallelism, optimized training kernels, and direct integration with production inference engines. The abstract reports up to 9.9x faster EAGLE-3 training for Qwen3-235B-A22B and up to 4.48x end-to-end inference speedup on SGLang using the released SpecBundle drafts. The practical contribution is a stack for producing and shipping draft models, not just a decoding idea in isolation. The biggest missing details are the hardware setup, baseline fairness, quality preservation, and how broadly the recipe generalizes beyond the named models and engine.

If the claims hold, the work could lower the operational barrier to adopting speculative decoding in real LLM serving pipelines. A key caveat is that the abstract does not say whether the speedups hold under fair baselines with comparable draft quality and serving settings. Another caveat is that reported gains may depend heavily on the specific combination of EAGLE-3, Qwen3-235B-A22B, and SGLang used in the abstract's examples. The abstract is underspecified on reproducibility details such as hardware, kernel configuration, and evaluation breadth.

### 6. cuGenOpt: A GPU-Accelerated General-Purpose Metaheuristic Framework for Combinatorial Optimization

*arXiv preprint | 2026-03-19T17:19:21Z | Yuyang Liu*

> This paper presents a CUDA-based metaheuristic framework for combinatorial optimization that tries to combine general-purpose modeling, GPU-parallel search, and extensibility in one system.

Combinatorial optimization systems usually trade off solver generality, raw performance, and ease of use, and this paper argues that GPUs can narrow that gap for a broader class of problems. The core design is a "one block evolves one solution" CUDA execution model, unified solution encodings, adaptive operator selection, hardware-aware resource management, and a registration path for custom CUDA operators, wrapped behind a Python JIT interface with an LLM-based modeling assistant. The abstract reports evaluations on five benchmark suites and three GPU architectures, claiming orders-of-magnitude gains over general MIP solvers, competitive quality against specialized solvers on instances up to n=150, and concrete improvements in TSP gap and VRPTW throughput from framework-level optimizations. If these claims hold, the framework could be useful as a systems substrate for GPU-native heuristic search rather than a single hand-tuned solver. The biggest missing detail is whether the baselines, operator implementations, and hardware-specific tuning are fair and reproducible enough to support the broad general-purpose claim.

The main value is a potentially reusable GPU optimization stack that could lower the cost of building fast heuristic solvers across multiple problem classes without rewriting each solver from scratch. A key caveat is that the paper is only loosely aligned with LLM inference or low-level code generation despite the CUDA focus. Another caveat is that the abstract bundles many claims, but it is unclear how much comes from fair baseline comparison versus framework customization and tuning effort per task.
