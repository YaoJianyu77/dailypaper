### 1. Calibrated Speculative Decoding: Frequency-Guided Candidate Selection for Efficient Inference

*ACL 2026 Main Conference | 2026-04-15T09:01:54Z | Xuwen Zhou, Fangxin Liu, Chao Wang, Xiao Zheng et al.*

> A training-free speculative decoding variant that tries to recover draft tokens rejected by exact-match verification through frequency-guided candidate rescue and probability-based acceptance.

This paper targets a known inefficiency in speculative decoding: standard verification can reject draft tokens that are semantically acceptable but lexically different, which limits realized speedup. The proposed Calibrated Speculative Decoding (CSD) adds two lightweight pieces, an Online Correction Memory that reuses recurring rejection patterns as rescue candidates and a Semantic Consistency Gating step that accepts candidates using probability ratios rather than exact token matching. The abstract claims gains across multiple LLMs, with peak throughput speedup up to 2.33x, while preserving accuracy and even improving performance on complex reasoning datasets. If those claims hold, the practical value is better speculative decoding efficiency without retraining, which is relevant for serving stacks that want more speed from existing model pairs. The main missing detail is that the abstract does not specify the verification cost, hardware/setup, baselines, or when semantic gating remains fair and stable versus standard exact verification.

For inference systems, the appeal is a low-overhead way to extract more throughput from speculative decoding without retraining models, if the added rescue and gating logic is cheaper than the rejection it avoids. A key caveat is that the abstract does not clarify whether the acceptance rule preserves the same decoding semantics as standard verification. Another caveat is that reported speedups may depend heavily on specific model pairs, task mixes, or hardware, which are not described here.

### 2. Event Tensor: A Unified Abstraction for Compiling Dynamic Megakernel

*arXiv preprint | 2026-04-14T22:19:51Z | Hongyi Jin, Bohan Hou, Guanjie Wang, Ruihang Lai et al.*

> This paper proposes a compiler abstraction for dynamic megakernels that aims to preserve the launch-overhead and synchronization benefits of persistent kernels while handling shape and data-dependent LLM inference workloads.

This paper targets a real systems bottleneck in LLM inference: kernel launch overheads and coarse synchronization can leave inter-kernel parallelism underused, while existing megakernel approaches become brittle once shapes or control flow are dynamic. It introduces Event Tensor, a compiler abstraction that represents dependencies among tiled tasks and is meant to support both shape-dependent and data-dependent dynamism inside a unified persistent-kernel compilation flow. On top of that abstraction, the Event Tensor Compiler applies static and dynamic scheduling transformations to generate high-performance persistent kernels for serving workloads. The abstract claims state-of-the-art LLM serving latency and substantially lower warmup overhead, which suggests value for deployment settings where dynamic behavior and startup cost both matter. The biggest missing detail is that the abstract does not say what workloads, hardware, baselines, or scheduling overheads were used to justify those claims.

If the abstraction and scheduler work as claimed, this could make dynamic LLM serving more compiler-manageable while retaining the latency benefits that usually require more rigid fused-kernel designs. A key caveat is that the abstraction may add runtime or compiler complexity that offsets part of the claimed latency benefit on some workloads. Another caveat is that the abstract does not reveal baseline fairness, so the state-of-the-art claim is hard to assess from the provided evidence alone.

### 3. YOCO++: Enhancing YOCO with KV Residual Connections for Efficient LLM Inference

*arXiv preprint | 2026-04-15T07:05:14Z | You Wu, Ziheng Chen, Yizhen Zhang, Haoyi Wu et al.*

> This paper modifies cross-layer KV sharing with bottom-layer residual links to recover capacity at fixed 50% KV-cache compression.

The paper targets a common tradeoff in LLM inference: cross-layer KV-cache compression reduces memory use but often hurts model quality. It extends YOCO, which reuses middle-layer KVs for upper layers, by adding weighted residual connections from each bottom-half layer to the bottom layer's KVs. The abstract claims this raises model capacity without changing training or inference efficiency, and reports state-of-the-art results among cross-layer KV compression methods at 50% KV-cache compression, including outperforming a standard Transformer. For systems readers, the practical value is a potentially better memory-quality point for long-context serving, but the abstract does not make the evaluation setup, hardware assumptions, or fairness of the baseline comparisons clear.

If the claims hold, the method could improve the memory-efficiency frontier for LLM inference by making KV-cache compression less accuracy-destructive at the same cache footprint. A key caveat is that the abstract gives no detail on hardware, latency measurements, or end-to-end serving throughput, so systems relevance is only partially established. Another caveat is that baseline fairness is unclear, especially for the claim of outperforming a standard Transformer under compression-aware conditions versus unconstrained settings more broadly for comparison quality and scope in the reported setup only at 50% compression rate and not other regimes or workloads in evidence provided here alone by the abstract summary itself only today in this input.

### 4. ToolSpec: Accelerating Tool Calling via Schema-Aware and Retrieval-Augmented Speculative Decoding

*arXiv preprint | 2026-04-15T06:05:11Z | Heming Xia, Yongqi Li, Cunxiao Du, Mingbo Song et al.*

> ToolSpec speeds up LLM tool calling by drafting schema-constrained outputs and reusing similar past tool invocations during speculative decoding.

This paper targets a practical serving bottleneck: multi-step tool calling adds substantial latency as LLM applications rely more heavily on structured external actions. The core claim is that tool-calling traces are structured enough to admit better speculative drafts, so ToolSpec combines schema-aware drafting with retrieval of similar historical invocations. Its mechanism alternates deterministic filling of schema-fixed tokens with speculative generation for variable fields, using a finite-state-machine style controller. The abstract reports benchmark results with up to 4.2x speedup over prior training-free speculative decoding methods, which makes the work directly relevant to inference-time serving efficiency. The main missing details are the hardware setup, benchmark mix, verification overhead, retrieval cost, and how robust the gains are when tool arguments are less repetitive or less schema-regular.

If the reported gains hold under realistic serving conditions, this offers a low-friction path to faster tool-augmented inference by exploiting structure already present in production tool schemas and call histories. A key caveat is that the reported gains may depend heavily on repetitive tool traces and strong schema regularity, which may not generalize across production workloads. Another caveat is that retrieval can add memory, indexing, and latency overhead that may offset decode-time savings in some deployments.

### 5. ATLAAS: Automatic Tensor-Level Abstraction of Accelerator Semantics

*arXiv preprint | 2026-04-15T06:15:56Z | Ruijie Gao, Haoran Jin, Jirong Yang, Nathaniel Bleier*

> This paper presents an MLIR pipeline that lifts RTL-derived bit-level accelerator semantics into tensor-level ISA specifications so compiler backends can be generated with much less manual ISA modeling.

This work targets a real bottleneck in accelerator evaluation: many tensor accelerators have RTL but lack usable ISA documentation and compiler backends, so software support remains narrow and expert-dependent. ATLAAS proposes an end-to-end MLIR lifting pipeline that starts from RTL-extracted bit-level LLVM IR and progressively reconstructs higher-level tensor semantics such as MAC patterns, saturation behavior, buffer structure, and layout transforms. The abstract reports substantial IR collapse on Gemmini, Z3-based equivalence checking, and successful reuse of the same pipeline on VTA without accelerator-specific changes. If the method is robust, the practical value is a shorter path from hardware RTL to a usable compiler stack for new accelerators. The main missing detail is how broadly this lifting works beyond the reported modules and how strong the evidence is for the claimed performance-competitive backend generation.

If the abstraction is reliable, it could reduce the manual compiler bring-up cost for new accelerators and make hardware-software co-design evaluation much more scalable. A key caveat is that the workflow depends on an upstream RTL-extraction step, so practical robustness may be limited by errors or information loss there. Another caveat is that the abstract shows only a small number of accelerator case studies, so generality across more complex designs remains uncertain.
