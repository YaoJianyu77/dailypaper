### 1. Cross-Session Threats in AI Agents: Benchmark, Evaluation, and Algorithms

*arXiv preprint | 2026-04-22T22:40:31Z | Ari Azarafrooz*

> This paper turns cross-session agent attacks into a benchmarked memory-selection problem and proposes a bounded-memory reader with a serving-aware stability metric.

This paper studies a failure mode in agent safety systems: session-local guardrails can miss attacks whose malicious payload is spread across many sessions and only becomes visible after aggregation. It introduces CSTM-Bench, a benchmark with executable cross-session attack taxonomies, matched benign confounders, and two shards designed to separate dilution from harder isolation-invisible cases. The abstract claims that both per-session judging and naive full-log long-context correlation lose roughly half their recall on the harder split, while a bounded-memory Coreset Memory Reader that keeps K=50 high-signal fragments is the only reader whose recall survives across both shards. The systems-relevant idea is CSR_prefix, which measures ordered prefix stability because fragment reshuffling can break KV-cache reuse, and combines that serving constraint with detection quality in the CSTM score. The main missing detail is external validity: the study is small, uses one correlator family without prompt optimization, and the abstract does not fully clarify baseline fairness or runtime cost tradeoffs.

If the method transfers, it gives agent builders a way to evaluate memory pipelines where security recall and serving stability are optimized together instead of treating long-context aggregation as a free fix. A key caveat is that the benchmark is modest in size and partly constructed, so real-world coverage is uncertain. Another caveat is that the evaluation uses one correlator family and no prompt optimization, leaving baseline strength and model dependence unresolved.

### 2. Distributed Generative Inference of LLM at Internet Scales with Multi-Dimensional Communication Optimization

*arXiv preprint | 2026-04-22T20:36:47Z | Jiu Chen, Shuangyan Yang, Xu Xiong, Hexiao Duan et al.*

> This paper presents BloomBee, a decentralized LLM inference system that attacks the internet-scale bandwidth bottleneck by jointly optimizing partitioning, batching, offloading, compression, and speculative decoding.

BloomBee targets decentralized LLM inference across heterogeneous internet nodes, where cross-node bandwidth rather than raw compute becomes the dominant systems bottleneck. The core claim is that communication must be optimized jointly, so the system combines LLM layer assignment, micro-batching, tensor offloading, lossless compression, and speculative decoding instead of tuning them in isolation. It formulates the coordination of these choices as an optimization problem and solves it with dynamic programming, which is the main mechanism described in the abstract. The reported evidence is an evaluation across multiple network environments with up to 1.76x higher throughput and up to 43.20% lower average latency than prior decentralized inference systems. The main missing details are the hardware and network assumptions, baseline fairness, per-component attribution, and the operational costs of the added coordination logic.

If the joint optimization is robust, the system could make bandwidth-limited, heterogeneous, internet-scale LLM serving materially more efficient than current decentralized approaches. A key caveat is that the abstract does not separate the effect of each optimization, so the true source of the gains is not yet clear. Another caveat is that internet-scale performance may depend on unstated assumptions about node stability, placement, and network variability that do not generalize cleanly to other deployments.

### 3. Sparse Forcing: Native Trainable Sparse Attention for Real-time Autoregressive Diffusion Video Generation

*arXiv preprint | 2026-04-23T02:22:25Z | Boxun Xu, Yuming Du, Zichang Liu, Siyu Yang et al.*

> The paper turns an observed persistent-block attention pattern in autoregressive video diffusion into a trainable sparse attention scheme and GPU kernel that reduce KV-cache cost while improving long-horizon generation quality.

Long-horizon autoregressive video diffusion becomes expensive because attention and KV-cache updates grow with the spatiotemporal history. The paper argues that rollouts repeatedly focus on a persistent subset of salient visual blocks, and uses that observation to build Sparse Forcing, which learns how to compress, preserve, and update those blocks while limiting each local window to a dynamic sparse neighborhood. It pairs this with Persistent Block-Sparse Attention (PBSA), a GPU kernel for sparse attention and memory updates during training and inference. The abstract reports VBench gains over Self-Forcing, 1.11-1.27x decoding speedups, and a 42% lower peak KV-cache footprint, with larger gains on longer 20-second and 1-minute rollouts. The main missing detail is whether these gains hold under clearly specified hardware, batch, resolution, and baseline-optimization conditions.

If the claimed sparse structure is robust and the kernel is practical, the method offers a path to lower-latency, lower-memory long-horizon generation by reducing both attention compute and cache-update overhead. A key caveat is that the reported speedups are moderate at short rollouts, so the added sparse-kernel complexity may only pay off clearly at longer horizons. Another caveat is that the abstract does not specify kernel portability, hardware assumptions, or whether the comparison used state-of-the-art dense attention implementations as baselines, which limits confidence in the systems claim from the abstract alone.

### 4. LightSeq: A High Performance Inference Library for Sequence Processing and Generation

*arXiv preprint | 2020-10-27 | Xiaohui Wang, Ying Xiong, Yang Wei, Mingxuan Wang et al.*

> This paper presents a CUDA inference library for transformer-family sequence models that combines coarse-grained fusion, dynamic memory reuse, and optimized autoregressive decoding to reduce serving overhead.

Transformer inference is often limited by kernel launch overhead, memory pressure, and token-by-token decoding costs, especially in sequence generation workloads. This paper presents LightSeq, a CUDA-based inference library for transformer-family models, and centers its design on coarse-grained fusion, dynamic memory reuse, and efficient autoregressive decoding. From the abstract, the contribution is a systems runtime/library effort rather than a new model, aimed at making sequence processing and generation run more efficiently on GPU. The practical value is clear if the implementation is solid: these mechanisms could improve latency and throughput for transformer serving. The biggest missing detail is evaluation evidence, since the abstract does not make the hardware setup, baselines, exact fused operations, or gain breakdown clear.

If the design works as claimed, it offers a useful systems reference for GPU transformer serving, especially around fusion boundaries, memory planning, and decode-loop efficiency. A key caveat is that the abstract does not say which parts of the stack are fused or how portable the fused kernels are. Another caveat is that performance claims cannot be assessed from the abstract because hardware, baselines, and gain breakdowns are missing or unclear.
