### 1. FleetOpt: Analytical Fleet Provisioning for LLM Inference with Compress-and-Route as Implementation Mechanism

*arXiv preprint | 2026-03-17T13:41:21Z | Huamin Chen, Xunzhuo Liu, Yuhan Liu, Junchen Jiang et al.*

> This paper treats LLM fleet sizing as an analytical queueing problem and pairs two-pool routing with gateway prompt compression to reduce KV-cache overprovisioning.

LLM serving fleets are often sized for rare long prompts, which strands GPU capacity in KV-cache slots that most requests never need. The paper formulates minimum-cost provisioning under a P99 TTFT target with an M/G/c queueing model and argues that the optimum is a two-pool design split by a context boundary B*. It then adds Compress-and-Route, where gateway-side extractive compression pushes borderline requests below that boundary and avoids the large cost jump of sending them to the long-context pool. The abstract reports 6-82% GPU cost reduction versus a homogeneous fleet on three production traces, plus simulator validation within 3% error on predicted utilization, with extra gains from compression beyond routing alone. The main missing details are whether compression preserves output quality, how realistic the serving-model assumptions are, and how sensitive the optimum is to workload drift and engine behavior.

If the analysis and compression tradeoff hold in practice, the method offers a fast planner for translating workload statistics and latency targets into lower-cost LLM fleet designs. A key caveat is that the reported savings may depend strongly on prompt-length skew and may shrink on workloads with flatter or shifting context distributions. Another caveat is that the abstract does not establish whether gateway compression preserves output quality, safety, or end-to-end latency under production constraints. The analytical optimum may be brittle if real serving behavior is dominated by batching policy, backpressure, or heterogeneous hardware effects rather than the modeled queueing regime.

### 2. VQKV: High-Fidelity and High-Ratio Cache Compression via Vector-Quantization

*arXiv preprint | 2026-03-17T12:16:06Z | Yixuan Wang, Qingyu Shi, Jiayu Zhou, Dianbo Liu et al.*

> A training-free vector-quantized KV-cache compression method that claims high compression with limited quality loss for long-context LLM inference.

This paper targets KV-cache growth as a deployment bottleneck for long-context LLM inference, especially when memory rather than compute limits usable context length. It proposes VQKV, a training-free compression method that replaces raw KV values with vector-quantized representations encoded by a small number of integer indices, aiming to improve the usual compression-versus-fidelity tradeoff. The abstract reports 82.8% KV-cache compression on LLaMA3.1-8B, 98.6% of baseline performance on LongBench, and 4.3x longer generation under the same memory footprint. If those numbers hold in realistic serving settings, the method could make long-context deployment materially more practical on constrained hardware. The biggest missing detail is the systems story: the abstract does not say how codebooks are built, what encode/decode overhead is added, or how fair the baselines and hardware settings are.

If runtime overhead is controlled, this could improve long-context serving capacity on memory-limited deployments by trading large KV buffers for compact index-based representations. A key caveat is that the abstract does not state encoding/decoding latency, bandwidth effects, or end-to-end throughput impact. Another caveat is that evaluation scope is unclear beyond LLaMA3.1-8B and LongBench, so generality is not established from the abstract alone.

### 3. Behavioral Steering in a 35B MoE Language Model via SAE-Decoded Probe Vectors: One Agency Axis, Not Five Traits

*arXiv preprint | 2026-03-17T10:05:41Z | Jia Qing Yap*

> This paper uses SAE-decoded probe directions to steer a 35B MoE model at inference time and argues that five intended behavioral traits mostly collapse into one dominant agency axis computed during prefill.

This paper studies inference-time behavioral steering in a 35B MoE language model by training sparse autoencoders on residual activations, fitting linear probes for five agentic traits, and decoding those probe weights back into continuous steering vectors. The core question is whether these behaviors can be independently controlled without retraining and where in the model's execution the relevant commitments are formed. The abstract claims that autonomy steering produces a large behavioral shift, tool-use steering has a smaller effect, and most of the apparent trait controls mostly reduce to a single agency axis rather than five cleanly separable dimensions. It also reports that steering during autoregressive decoding has no effect, which points to prefill as the stage where behavioral decisions are set in this GatedDeltaNet-based architecture. The practical systems value is therefore more about locating control points in inference than about efficient serving, and the abstract leaves key checks unclear, including intervention placement, baseline fairness, and robustness across prompts and rollout settings.

If the claims hold, the main value is a mechanistic handle on when behavioral control enters the forward pass of hybrid recurrent-attention LMs and a practical way to test inference-time steering hypotheses without model retraining. A key caveat is that this is only loosely aligned with inference acceleration or kernel/runtime co-design, so the systems payoff is indirect. Another caveat is that the abstract does not specify enough about intervention sites, prompt distribution, or baseline fairness to judge reproducibility or causal strength fully.

### 4. Residual Stream Duality in Modern Transformer Architectures

*arXiv preprint | 2026-03-17T00:56:29Z | Yifan Zhang*

> This paper reframes the residual stream as a depth-axis analogue of local causal attention and argues that operator duality does not imply equal systems behavior.

This paper argues that Transformer design is easier to reason about if sequence position and layer depth are treated as two ordered axes, with the residual stream viewed as a representational pathway rather than simple additive plumbing. Its central claim is that, for a fixed token, causal depth-wise residual attention is the same local operator as causal short sliding-window attention, but written over depth instead of sequence. Using that lens, the paper positions prior depth-aggregation and vertical-attention work, then argues that the apparent symmetry breaks at the systems level because sequence-axis ShortSWA can better reuse sliding-window kernels, KV-cache layouts, and chunked execution in large autoregressive models. The practical recommendation is to use sequence-axis ShortSWA for local adaptive mixing and Deep Delta Learning (DDL) when the goal is to modify the shortcut itself. The main missing detail is that the abstract does not make clear whether the paper adds new experiments or mainly offers a conceptual synthesis, so the empirical strength of the claims is unclear.

If the framing is sound, it helps practitioners separate shortcut-modification ideas from local-mixing ideas and prefer placements that align with existing kernels, KV-cache layouts, and chunked autoregressive execution. A key caveat is that the abstract reads partly as a conceptual synthesis, so the amount of new method or empirical contribution is unclear. Another caveat is that the systems argument is high level and does not include concrete latency, throughput, memory, or baseline details in the abstract.

### 5. Dimensional Type Systems and Deterministic Memory Management: Design-Time Semantic Preservation in Native Compilation

*arXiv preprint | 2026-03-17T12:19:07Z | Houston Haynes*

> A semantics-heavy compiler framework that keeps dimensional type information alive through MLIR lowering so numeric representation and deterministic allocation can be chosen from one shared graph.

This paper argues that native compilation usually erases semantic information too early, which makes representation sizing and memory placement less principled. It proposes a Dimensional Type System that persists through multi-stage MLIR lowering and feeds a program semantic graph where range inference, word width, memory footprint, and allocation strategy are resolved together. The abstract claims polynomial-time, complete, principal inference and a deterministic memory discipline based on four escape classes, and it also sketches compatibility with forward-mode autodiff. If this works in practice, it could make low-level representation and memory decisions inspectable at design time instead of pushing them into later ad hoc passes. The main missing detail is practical evidence: the abstract does not make implementation maturity, backend coverage, or any runtime or code-quality gains clear.

If the framework holds up, it offers compiler builders a unified semantic basis for representation-width and allocation decisions, with more immediate value for compiler correctness and inspectability than for direct inference-serving speedups. A key caveat is that the abstract reads primarily as a PL and compiler-semantics paper, so its relevance to LLM inference systems or CUDA kernel generation is indirect. Another caveat is that no evaluation setup, target hardware, or baseline comparison is stated, so practical impact on performance or code quality is unproven from the abstract alone.

### 6. An Efficient Heterogeneous Co-Design for Fine-Tuning on a Single GPU

*arXiv preprint | 2026-03-17T12:05:17Z | Ruijia Yang, Zeyi Wen*

> SlideFormer is a single-GPU LLM fine-tuning system that combines asynchronous GPU/CPU/I/O overlap, heterogeneous memory management, and Triton kernel tuning to push much larger models onto commodity hardware.

This paper targets the memory and data-movement bottlenecks that make LLM fine-tuning hard to run on a single consumer GPU. SlideFormer treats the GPU as a sliding compute window, overlaps GPU work with CPU updates and multi-tier I/O, and adds a heterogeneous memory scheme plus optimized Triton kernels for bottleneck paths. The abstract claims fine-tuning of 123B+ models on one RTX 4090, support for up to 8x larger batch sizes and 6x larger models than baselines, 1.40x to 6.27x higher throughput, roughly half the CPU/GPU memory usage, and more than 95% peak performance on both NVIDIA and AMD GPUs. If those claims hold under fair comparisons, the practical value is in end-to-end runtime and memory-system co-design rather than a single isolated trick. The biggest missing detail is how much of the gain comes from kernels versus scheduling and offloading, and the abstract does not make the baseline setup or fairness checks clear.

If reproducible, this could lower the hardware barrier for practical LLM adaptation and provide a useful design pattern for memory-aware runtime and kernel co-design on constrained systems. A key caveat is that the abstract does not identify the baselines or isolate which subsystem contributes most of the reported improvement. Another caveat is that single-GPU feasibility may depend heavily on host memory, storage bandwidth, and the exact fine-tuning recipe, none of which are specified here clearly enough for replication judgment from the abstract alone.
