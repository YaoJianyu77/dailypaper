### 1. Goose: Anisotropic Speculation Trees for Training-Free Speculative Decoding

*arXiv preprint | 2026-04-02T13:48:42Z | Tao Jin, Phuong Minh Nguyen, Naoya Inoue*

> This paper argues that training-free speculative decoding should use an asymmetric token tree when draft sources have very different acceptance rates, and implements that idea as an adaptive spine-plus-branches structure.

The paper studies a systems bottleneck in speculative decoding: under a fixed verification budget, tree depth and tree breadth trade off against each other, and balanced trees waste budget when draft sources have very different acceptance quality. Its key observation is that context-matched n-gram tokens and prior-pass statistical predictions have a large acceptance-rate gap, so they should not be placed symmetrically in the same draft tree. Goose addresses this with an anisotropic speculation tree: high-confidence context-matched tokens form a deep spine, while lower-confidence alternatives branch out more widely from each spine node. The abstract provides both a theoretical claim, namely that this structure accepts at least as many tokens per step as either source alone, and empirical results across five models and five benchmarks with lossless speedups and gains over balanced-tree baselines under the same budget. The main missing detail for a full read is how adaptive tree construction, verification cost accounting, and baseline fairness are implemented in practice.

If the method is robust, it offers a practical serving-side improvement for speculative decoding by turning heterogeneous draft quality into better latency-throughput tradeoffs without extra training. A key caveat is that the method may depend strongly on having reliable context-matched tokens, which could limit gains on low-overlap or highly novel generations. Another caveat is that the abstract does not specify runtime overheads, hardware assumptions, or implementation complexity, so net serving benefit still needs checking beyond token acceptance theory.

### 2. FlatAttention: Dataflow and Fabric Collectives Co-Optimization for Large Attention-Based Model Inference on Tile-Based Accelerators

*Submitted to IEEE Transactions on Computers. 14 pages | 2026-04-02T14:45:19Z | Chi Zhang, Luca Colagrande, Renzo Andri, Luca Benini*

> This paper presents an attention dataflow for tile-based accelerators that trades HBM traffic for on-chip fabric collectives to raise utilization and improve large-model inference throughput.

The paper targets a systems shift where attention is becoming a larger inference bottleneck, especially for MoE decoding on chip-scale and wafer-scale tile-based accelerators. It proposes FlatAttention, a dataflow that restructures attention execution around collective primitives in the on-chip fabric so more data movement stays on chip and expensive HBM accesses are reduced. The abstract reports strong kernel- and system-level gains, including higher utilization, lower HBM traffic, speedups over FlashAttention-3 and GH200-based implementations, and end-to-end decoding improvements for DeepSeek-v3 FP8. For this audience, the main interest is the co-design angle between attention scheduling and the accelerator interconnect rather than a generic attention kernel optimization. The biggest missing detail is how fair the hardware comparison, baseline implementation quality, and generalization claims are across attention variants and deployment settings.

If the claims hold, this gives a useful template for compiler-runtime-hardware co-design on tile-based LLM inference systems where communication primitives are first-class optimization levers. A key caveat is that the strongest gains may depend on a specific tile fabric and may not transfer to more conventional GPU systems. Another caveat is that the abstract does not clarify how optimized the comparison baselines are, especially against FlashAttention-3 and GH200 implementations.

### 3. From Threads to Tiles: T2T, a Compiler for CUDA-to-NPU Translation via 2D Vectorization.

*CGO | 2026 | Shuaijiang Li, Jiacheng Zhao, Ying Liu 0055, Shuoming Zhang et al.*

> This paper presents a compiler that lifts CUDA's thread-level SIMT structure into analyzable loop nests and then maps selected loop pairs onto NPU-style 2-D tile instructions.

This paper targets the mismatch between CUDA's fine-grained thread model and emerging NPUs that execute coarse-grained 2-D tile operations. It proposes T2T, a compiler pipeline that first rewrites implicit SIMT parallelism into an explicit loop representation through a Unified Parallelism Abstraction, then applies 2-D vectorization to map loop pairs onto NPU tile instructions. The abstract claims this translation is combined with semantics-preserving handling of control flow and warp-level intrinsics, which matters because those features often block direct retargeting of CUDA kernels. Reported evidence is meaningful but limited to the abstract: the implementation is built on Polygeist, reaches up to 73% of native CUDA performance on an A100-based comparison point, and beats baseline translation methods by up to 6.9x on representative NPUs. The main missing detail is evaluation transparency, especially which kernels transfer well, what the NPU hardware assumptions are, and how fair the baselines are.

If the method generalizes, it offers a concrete compiler path for reusing CUDA kernel code on tile-based NPUs without fully rewriting kernels by hand. A key caveat is that the reported 73% of native CUDA performance uses A100 as the reference point, which may not translate into a clear practical portability story without more apples-to-apples detail. Another caveat is that the abstract does not clarify whether the method depends on a narrow class of affine or regular kernels, which could limit usefulness for real CUDA workloads with irregular control or memory access patterns.

### 4. GPU-RMQ: Accelerating Range Minimum Queries on Modern GPUs

*arXiv preprint | 2026-04-02T09:23:08Z | Lara Kreis, Justus Henneberg, Valentin Henkys, Felix Schuhknecht et al.*

> GPU-RMQ proposes a hierarchical GPU range-minimum-query index that mixes CUDA cores and ray-tracing cores to cut memory and build cost while improving query throughput on large arrays.

This paper targets GPU range minimum queries, a primitive used in string processing and database workloads, where prior GPU methods appear to trade off badly between memory footprint, index build cost, and throughput on large inputs. The proposed system builds a hierarchy of range-minimum summaries in parallel, then answers queries by scanning only the relevant hierarchy regions with an optimized massively parallel operation. A hybrid execution design uses both ray-tracing cores and CUDA cores at different hierarchy levels, which is the main systems mechanism behind the claimed efficiency gains. The abstract reports higher throughput than prior GPU approaches, much lower memory use, and dramatically faster index construction, especially on larger arrays. The main missing detail is how much of the gain comes from the hierarchy itself versus the hybrid hardware mapping, and whether the comparison setup and workload mix are fair across methods.

If the claims hold, this looks like a useful GPU data-structure design pattern for large-query workloads where preprocessing cost and memory capacity matter as much as raw query speed. A key caveat is that this is not directly an LLM inference paper, so relevance to the core reading focus is indirect. Another caveat is that the abstract does not separate the benefit of the hierarchy from the benefit of specialized RTX hardware usage, which may limit portability conclusions.

### 5. A Case For Host Code Guided GPU Data Race Detector

*arXiv preprint | 2026-04-02T14:35:49Z | Ajay Nayak, Anubhab Ghosh, Arkaprava Basu*

> This paper argues that GPU race detection improves materially when static analysis uses host-side launch code to constrain which kernel parameter values can actually occur.

This paper targets a familiar GPU tooling gap: dynamic race detectors miss bugs that do not trigger in tests, while static analyzers often over-report because they reason over impossible parameter values. The core idea is to analyze host launch code together with GPU kernel code so the detector can use CPU-side semantic constraints on kernel arguments. The abstract claims this produces a static detector, HGRD, that covers a broad set of real races while eliminating the false alarms seen in prior static tools, and that it also avoids the misses of prior dynamic tools. For systems readers, the value is less about LLM inference directly and more about making generated or hand-written GPU kernels safer without relying on test coverage alone. The biggest missing detail is evaluation context: the abstract does not make clear the benchmark suite, supported GPU programming model details, analysis cost, or how broadly the zero-false-alarm claim holds.

If the claims hold, this could make GPU kernel development and generated CUDA-style code more debuggable by improving pre-deployment race detection precision without relying on test-triggered failures. A key caveat is that the abstract makes unusually strong accuracy claims, but does not expose the dataset, corner cases, or failure modes needed to judge robustness. Another caveat is that the contribution is a program analysis result rather than an inference or compiler-speedup result, so its relevance to LLM systems is indirect unless you care about GPU kernel correctness tooling.

### 6. LLM-VeriOpt: Verification-Guided Reinforcement Learning for LLM-Based Compiler Optimization.

*CGO | 2026 | Xiangxin Fang, Jiaqin Kang, Rodrigo Rocha, Sam Ainsworth 0001 et al.*

> This paper trains a small LLM for LLVM IR peephole optimization with verifier-driven reinforcement learning so correctness signals from Alive2 become part of the optimization loop.

This paper targets a central blocker in LLM-based compiler optimization: generated low-level rewrites are often invalid, so raw model capability is not enough for safe optimization. The authors propose LLM-VeriOpt, which trains Qwen-3B with GRPO using Alive2 semantic-equivalence feedback in the reward, aiming to make the model learn to self-correct toward verifiable LLVM IR transformations. The abstract claims substantially higher modification success, 90% verifiably correct outputs, and speedups over O0 code that are comparable to LLVM instcombine, with some emergent cases exceeding the handwritten pass. For systems readers, the interesting point is not just code generation but compiler-runtime co-design between an LLM policy and a formal verifier. The biggest missing detail from the abstract is the evaluation setup: it does not make clear the benchmark mix, verifier cost, training data construction, or how fair the baselines are.

If the claims hold, this suggests a practical path for pairing small LLMs with formal verification to produce safer low-level optimizers without relying only on larger general-purpose models. A key caveat is that the abstract is strong on headline results but weak on benchmark detail, so baseline fairness is still unverified. Another caveat is that reported optimization gains may depend heavily on the chosen peephole tasks and may not generalize to broader compiler phases or architectures.
