### 1. Where Matters More Than What: Decoding-aligned KV Cache Compression via Position-aware Pseudo Queries

*arXiv preprint | 2026-03-12T05:36:32Z | Zhenxu Tian, Yi Su, Juntao Li, Min Zhang*

> This paper proposes a KV-cache eviction method that approximates decoding-time attention with position-aware pseudo queries instead of relying on prompt-side prefill observations.

KV-cache compression for long-context LLM inference usually scores token importance from prompt-side attention during prefill, which can miss the tokens that matter later in decoding. This paper argues that the observation window should be aligned with future decoding queries and proposes DapQ, which builds position-aware pseudo queries to approximate those otherwise unavailable queries and guide eviction. The claimed gain comes from retaining tokens that generation will actually attend to, so the contribution is a better cache policy rather than a model change. The abstract reports superior results across multiple benchmarks and LLMs, including nearly lossless 99.5% performance on NIAH with a 3% KV-cache budget. What still needs checking is the construction cost of these pseudo queries, the fairness of the baseline comparison, and whether the position-dominant assumption holds beyond the reported tasks.

If the decode-aligned proxy is cheap and robust, it could improve long-context serving efficiency by reducing KV memory while preserving generation quality better than prompt-only heuristics. A key caveat is that the abstract does not specify pseudo-query overhead, so net end-to-end serving benefit is still uncertain. Another caveat is that the strongest stated result is on NIAH, and transfer to broader open-ended generation workloads is not clear from the abstract alone.

### 2. IndexCache: Accelerating Sparse Attention via Cross-Layer Index Reuse

*arXiv preprint | 2026-03-12T17:27:21Z | Yushi Bai, Qian Dong, Ting Jiang, Xin Lv et al.*

> The paper cuts sparse-attention overhead by reusing top-k token indices across neighboring layers instead of running a quadratic indexer at every layer.

This paper targets a practical bottleneck in production sparse attention for long-context LLM serving: even when core attention is reduced to O(Lk), the token indexer can remain O(L^2) and must normally run at every layer. The main idea is to exploit cross-layer similarity in top-k selections by keeping indexers only in a small set of Full layers and letting Shared layers reuse the nearest retained indices; the paper studies both a training-free greedy layer selection method and a training-aware distillation variant. In the abstract, the reported evidence is a 30B DSA model where 75% of indexer computations are removed with negligible quality degradation, yielding up to 1.82x prefill speedup and 1.48x decode speedup, plus preliminary confirmation on GLM-5. If these results hold under realistic serving conditions, the method is a clean systems lever because it attacks index construction cost directly rather than only optimizing the sparse attention kernel. The main missing detail is how robust the reuse pattern is across workloads, layer layouts, hardware setups, and baseline implementations, since the abstract does not make those evaluation conditions clear.

If the reuse assumption is robust, this offers a practical way to lower sparse-attention serving cost by removing redundant index work in both prefill and decode. A key caveat is that the gains may rely on strong cross-layer similarity that could weaken on other models, prompts, or longer contexts. Another caveat is that the abstract does not specify baseline fairness, hardware conditions, or implementation details, so the reported speedups are hard to normalize against other systems work.

### 3. LongFlow: Efficient KV Cache Compression for Reasoning M

*arXiv preprint | 2026-03-12T03:46:35Z | Yi Su, Zhenxu Tian, Dan Qiao, Yuechi Zhou et al.*

> A KV cache compression method for long-output reasoning models that pairs query-only importance scoring with a fused attention-and-eviction kernel to cut memory and bandwidth costs during decoding.

LongFlow targets the long-output regime of reasoning models, where KV cache growth becomes a dominant decode-time memory and bandwidth bottleneck. The paper proposes an importance metric computed from an intermediate attention result using only the current query, avoiding the repeated expensive scoring used by earlier KV compression methods. It also adds a custom fused operator that combines FlashAttention, importance estimation, and token eviction, so the systems story is not just better token selection but lower runtime overhead. The abstract reports up to 11.8x throughput improvement at 80% KV cache compression with minimal accuracy impact. The main missing detail is that the abstract does not make the hardware setup, baselines, workload mix, or contribution split between compression policy and kernel fusion clear.

If the claims hold, this could materially reduce serving cost for long-chain reasoning models by lowering KV residency and decode-time bandwidth pressure without adding much extra compute. A key caveat is that the abstract does not separate gains from better eviction decisions and gains from custom kernel fusion. Another caveat is that accuracy impact is only described qualitatively, with no task-level breakdown or failure cases in the abstract given here.

### 4. MobileKernelBench: Can LLMs Write Efficient Kernels for Mobile Devices?

*arXiv preprint | 2026-03-12T13:48:11Z | Xingze Zou, Jing Wang, Yuhua Zheng, Xueyi Chen et al.*

> A mobile-kernel benchmark plus a repository-aware multi-agent generator shows that generic LLMs mostly fail to compile on MNN CPU, while a grounded agent can produce some kernels that measurably outperform native libraries.

This paper studies whether LLMs can generate efficient kernels for mobile devices, a setting that is harder than generic code generation because frameworks are fragmented and correctness must be checked on device. It introduces MobileKernelBench, which combines diverse operators, cross-framework interoperability, and an automated host-device pipeline for mobile kernel evaluation. On the CPU backend of MNN, the authors report that standard and fine-tuned LLMs have compilation failure rates above 54% and produce negligible performance gains, mainly due to hallucinations and weak framework grounding. To address that, they propose MoKA, a repository-aware multi-agent plan-and-execute system, and claim 93.7% compilation success with 27.4% of generated kernels achieving measurable speedups over native libraries. The main missing details in the abstract are the hardware setup, the size of those speedups, and whether the gains transfer beyond MNN CPU, so the systems impact is promising but still bounded.

If the claims hold, the work provides both a more realistic benchmark and a grounded generation recipe for low-level code synthesis on constrained runtimes, though its immediate relevance is stronger for mobile CPU stacks than for CUDA-centric inference systems. A key caveat is that the evidence in the abstract is centered on the MNN CPU backend, so generalization to other mobile frameworks, NPUs, or GPU kernels is unproven. Another caveat is that only 27.4% of generated kernels are reported to beat native libraries, which suggests limited end-to-end practical win even after the agentic improvements are applied to compilation success alone.

### 5. NCCLbpf: Verified, Composable Policy Execution for GPU Collective Communication

*arXiv preprint | 2026-03-12T02:03:55Z | Yusheng Zheng*

> This paper turns NCCL plugins into a verified, hot-reloadable policy layer by embedding a userspace eBPF runtime, aiming to keep collective tuning flexible without native-code failure risk.

Large-scale GPU communication stacks rely on NCCL plugins for tuning, but those plugins currently execute as unverified native code inside NCCL and can crash jobs or corrupt state. This paper introduces NCCLbpf, which embeds a userspace eBPF runtime into existing NCCL plugin interfaces so policies can be statically verified at load time, coordinate through structured maps, and be hot-reloaded atomically without modifying NCCL itself. The abstract reports very low runtime cost on 8x NVIDIA B300 GPUs over NVLink, rejection of all tested unsafe plugin behaviors, and up to 27% higher AllReduce throughput from a message-size-aware policy relative to NCCL default for 4-128 MiB messages. The practical appeal is safer and more composable collective tuning for production GPU systems, especially where policy changes currently require risky native plugins or downtime. The biggest missing detail is how restrictive the verifier is and whether the gains hold beyond the shown hardware, topology, and default-baseline comparison.

If the design generalizes, it offers a safer way to deploy and update collective-communication policies in production GPU stacks with little steady-state overhead. A key caveat is that the reported speedup is shown for one message-size-aware policy on one 8x B300 NVLink setup, so generality is unclear. Another caveat is that the main performance baseline in the abstract is NCCL default rather than a hand-tuned native plugin, which leaves baseline fairness open.

### 6. DeepSpeed Inference: Enabling Efficient Inference of Transformer Models at Unprecedented Scale

*SC22: International Conference for High Performance Computing, Networking, Storage and Analysis | 2022-06-30 | Reza Yazdani Aminabadi, Samyam Rajbhandari, Minjia Zhang, Ammar Ahmad Awan et al.*

> A transformer serving system that combines multi-GPU execution, kernel tuning, and CPU/GPU/NVMe memory offload to run larger models within latency and throughput budgets.

This paper targets transformer inference at scales where GPU memory limits, communication overhead, and kernel efficiency all become serving bottlenecks. It proposes an integrated inference stack that combines multi-GPU execution, inference-oriented kernel optimizations, and a heterogeneous CPU/GPU/NVMe memory hierarchy. From the abstract, the main claim is that this combination enables larger-scale serving while respecting latency and throughput constraints, but it does not specify how much each component contributes or under what hardware setup. If the design works as implied, it is practically useful as a serving blueprint for larger-than-memory transformer deployments. The biggest missing detail is the evaluation story: the abstract does not make the baselines, workloads, fairness, or tail-latency tradeoffs clear.

If the system behaves as claimed, it offers a practical way to trade compute, communication, and storage hierarchy capacity to serve models beyond single-GPU memory limits. A key caveat is that the abstract does not isolate which subsystem actually drives the claimed benefits. Another caveat is that results may depend heavily on hardware topology and communication characteristics that are not specified here.
