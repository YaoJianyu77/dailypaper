### 1. AQPIM: Breaking the PIM Capacity Wall for LLMs with In-Memory Activation Quantization

*arXiv preprint | 2026-04-20T12:04:51Z | Kosuke Matsushima, Yasuyuki Okoshi, Masato Motomura, Daichi Fujiki*

> This paper proposes in-memory product quantization of LLM activations and KV cache so PIM can hold more context and run attention with less interconnect traffic.

The paper starts from a clear systems bottleneck: long-context LLM decoding can overflow limited PIM capacity because KV cache growth dominates memory usage, while locality-breaking methods such as sparse attention do not fit PIM well. It argues that activation-side quantization should be redesigned for PIM rather than borrowed from generic quantization pipelines, and introduces AQPIM, a PIM-aware product-quantization framework that quantizes in memory and computes attention directly on compressed data. The abstract claims this cuts memory footprint, reduces attention-side compute overhead, and sharply lowers GPU-CPU communication, which it says can account for 90 to 98.5% of decoding latency. It also reports a 3.4x speedup over a state-of-the-art PIM approach. The main missing details are the hardware setup, model and context regimes, baseline fairness, and the actual accuracy cost of the quantization.

If the results hold, this gives a concrete hardware-software co-design path for making PIM more viable for long-context LLM serving by turning activation compression into a capacity, bandwidth, and latency optimization. A key caveat is that the abstract does not provide concrete accuracy results, so the compression-quality tradeoff is still unresolved. Another caveat is that the reported gains may depend strongly on a particular heterogeneous GPU-CPU-PIM system and communication profile that may not generalize widely given only the abstract.

### 2. Latent Phase-Shift Rollback: Inference-Time Error Correction via Residual Stream Monitoring and KV-Cache Steering

*Under Review | 2026-04-20T17:53:33Z | Manan Gupta, Dhruv Kumar*

> This paper proposes an inference-time control method that watches a model's residual stream for error-like phase shifts, then rolls back the KV cache and applies a steering vector to redirect generation.

This paper targets a practical failure mode in autoregressive reasoning: once a model takes a wrong intermediate step, later tokens usually amplify the mistake instead of repairing it. The proposed method, Latent Phase-Shift Rollback, monitors the residual stream at a selected layer, detects abrupt directional reversals with a cosine-similarity plus entropy gate, and then rolls back the KV cache while injecting a pre-computed steering vector, all without fine-tuning, gradients, or extra forward passes. The abstract reports strong gains on MATH-500 for an 8B model over standard autoregressive decoding, prompted self-correction, and Best-of-16, and it also includes a layer sweep showing that the best layer for error detection is not the best layer for correction. If the method is robust, it points to a lightweight inference-time intervention path that sits between plain decoding and much more expensive search-based correction. The main missing details are deployment cost, how stable the steering vector and trigger thresholds are across tasks and models, and whether the gains extend beyond this reasoning benchmark.

For LLM inference work, the value is a potentially cheap intervention mechanism that uses internal signals and KV-cache control rather than extra search or retraining to improve reasoning trajectories. A key caveat is that the evidence in the abstract is concentrated on MATH-500, so generalization to broader workloads is unproven. Another caveat is that the method may depend heavily on layer choice, trigger thresholds, and steering-vector quality, which could make it brittle in deployment.

### 3. HybridGen: Efficient LLM Generative Inference via CPU-GPU Hybrid Computing

*arXiv preprint | 2026-04-20T17:25:44Z | Mao Lin, Xi Wang, Guilherme Cox, Dong Li et al.*

> The paper proposes a long-context LLM inference runtime that computes attention jointly on CPU and GPU, using tiered-memory-aware KV placement instead of treating the CPU as a passive offload target.

This paper targets a central long-context inference bottleneck: KV caches grow large enough to overwhelm GPU memory capacity and bandwidth, while existing pruning or offloading schemes leave CPU compute and expanded memory underused. It proposes HybridGen, a CPU-GPU collaborative attention framework for tiered-memory systems such as CXL-expanded setups, built around attention logit parallelism, a feedback-driven scheduler, and semantic-aware KV cache mapping. The abstract claims experiments across three LLM families, eleven model sizes, and three GPU platforms with CXL-expanded memory, where HybridGen beats six prior KV-cache management methods by 1.41x to 3.2x on average while preserving better accuracy. If the method generalizes, the value is a more balanced way to trade compute, bandwidth, and memory locality for long-context serving. The biggest missing detail is how much of the gain depends on the specific CXL/tiered-memory hardware setup, scheduler overheads, and the fairness of the baseline configurations.

If the claims hold beyond the tested platforms, the work offers a concrete systems path to extend long-context serving by jointly exploiting heterogeneous compute and larger memory tiers rather than relying only on pruning or offload. A key caveat is that the reported gains may rely heavily on CXL-expanded memory and may not transfer cleanly to standard CPU-GPU servers. Another caveat is that the abstract does not reveal whether CPU participation increases end-to-end latency variance or introduces scheduler overhead that matters at smaller contexts or batches.

### 4. Copy-as-Decode: Grammar-Constrained Parallel Prefill for LLM Editing

*arXiv preprint | 2026-04-20T12:29:53Z | Ziyang Liu*

> The paper turns copy-heavy LLM editing into grammar-constrained decoding with explicit copy spans and replaces token-by-token copying with parallel-prefill KV-cache updates.

LLM editing is wasteful because standard autoregressive decoding regenerates the full output even when most edited tokens are copied verbatim from the input. This paper reframes editing as structured decoding over `<copy>` spans and `<gen>` regions, then executes copy spans with a single parallel-prefill forward instead of many decode steps, reusing the kernel shape of speculative decoding while relying on program constraints rather than probabilistic acceptance. The abstract supports this mainly with an upper-bound analysis rather than a deployed end-to-end system: it reports 6.8x-303x kernel speedups for copying, 74-98% line-level oracle copy coverage on ProbeEdit and HumanEvalPack-Fix, and lossless round-trips on 482 oracle programs. A small fine-tuning pilot suggests the representation is learnable, but the paper explicitly does not claim a production-ready selector, and the perturbation study shows strong brittleness to span-boundary errors. The main missing detail is whether end-to-end span selection, batching, and multi-file editing preserve the claimed latency gains under fair serving baselines.

If the selector and serving integration work end to end, this could shift copy-heavy editing from decode-bound execution toward prefill-bound execution, which is directly relevant to low-latency code and text editing systems. A key caveat is that most of the evidence is oracle-based or upper-bound analysis, so realized production gains are still unproven. Another caveat is that the method appears highly sensitive to span-boundary mistakes, which makes selector quality the dominant practical risk for end-to-end deployment.

### 5. River-LLM: Large Language Model Seamless Exit Based on KV Share

*Accepted to ACL 2026, 13pages, with appendix | 2026-04-20T15:20:17Z | Yingtao Shen, An Zou*

> A training-free early-exit framework for decoder-only LLMs that shares KV state through an auxiliary exit path and claims real wall-clock speedups instead of only theoretical layer savings.

River-LLM targets the main practical failure mode of early exit in decoder-only LLMs: once layers are skipped, later tokens no longer have the KV history those layers would have produced, so nominal compute savings do not cleanly turn into latency gains. The paper proposes a training-free "KV-Shared Exit River" that generates and preserves the missing KV states during token-level exits, avoiding recomputation or masking-based recovery. It also uses state-transition similarity inside decoder blocks to estimate cumulative KV error and choose exits more precisely. The abstract reports experiments on mathematical reasoning and code generation tasks, with claimed 1.71-2.16x practical speedup while keeping generation quality high. The main missing details are the hardware/setup assumptions, baseline fairness, and how stable the error-guided exit policy is across models and prompt distributions.

If the method holds up, it offers a deployment-friendly way to convert layer skipping into real latency reduction for existing decoder-only LLMs without retraining. A key caveat is that the abstract does not disclose the evaluation setup, so the reported speedups are hard to map to real serving conditions. Another caveat is that high generation quality" is underspecified and may hide task-specific or model-specific regressions. The abstract does not make those tradeoffs clear in detail.

### 6. GPTQ: Accurate Post-Training Quantization for Generative Pre-trained Transformers

*arXiv preprint | 2022-10-31 | Elias Frantar, Saleh Ashkboos, Torsten Hoefler, Dan Alistarh*

> A post-training quantization method for GPT-style models that claims to preserve accuracy at 3-4-bit weight precision and make much larger models deployable on limited hardware.

GPTQ tackles the deployment problem that large GPT-style models are hard to run on limited hardware because model weights are expensive to store and move. The paper proposes a scalable post-training quantization method whose main claim is that GPT-style models can be pushed to 3-4-bit weight precision while preserving accuracy. If that claim holds, the practical payoff is straightforward: much larger models become deployable under tighter memory budgets without a retraining-heavy pipeline. The abstract supports that headline at a high level, but it does not explain the quantization mechanism in enough detail to show where the robustness comes from. It also leaves the evaluation setup, hardware assumptions, and baseline comparisons unclear, which are the main items to verify in a full read.

If the claim holds, the method offers a practical path to cut model memory footprint and widen deployment of large generative models on constrained hardware. A key caveat is that the abstract does not explain the mechanism well enough to show why accuracy is preserved at such low precision. Another caveat is that evaluation details, hardware assumptions, and baseline fairness are not specified in the abstract, so the strength of the claim is hard to judge from the summary alone.
