### 1. VitaLLM: A Versatile, Ultra-Compact Ternary LLM Accelerator with Dependency-Aware Scheduling

*arXiv preprint | 2026-04-30T04:07:21Z | Zi-Wei Lin, Tian-Sheuan Chang*

> VitaLLM is a 16nm ternary LLM inference accelerator that combines heterogeneous compute cores, KV-fetch pruning, and dependency-aware scheduling for edge decoding.

Edge LLM deployment is constrained by memory bandwidth and power, and ternary models reduce storage while introducing workload imbalance and dependency issues on general-purpose hardware. VitaLLM proposes a hardware-software co-designed accelerator for ternary LLM inference, splitting large ternary projections and mixed-precision attention across specialized TINT-Cores and a unified BoothFlex-Core. It adds Leading One Prediction to prune redundant KV-cache fetches and dependency-aware scheduling to hide nonlinear-operation latency. The abstract reports a TSMC 16nm implementation with 70.70 tokens/s decoding throughput, 0.223 mm^2 area, 65.97 mW power, and 17.4 TOPS/mm^2/W FOM, with a claim of outperforming prior accelerators. The biggest missing detail is the evaluation setup: model sizes, sequence lengths, baselines, workload mix, and comparison methodology are not specified in the abstract.

If the reported design generalizes beyond the stated setup, it could be useful for studying ASIC co-design points for low-power ternary LLM serving rather than GPU-oriented CUDA acceleration. A key caveat is that the abstract does not specify the evaluated LLMs, sequence lengths, or batching assumptions. Another caveat is that the comparison to state-of-the-art accelerators may depend heavily on process, precision, memory hierarchy, and workload normalization details not given here.

### 2. TwinGate: Stateful Defense against Decompositional Jailbreaks in Untraceable Traffic via Asymmetric Contrastive Learning

*arXiv preprint | 2026-04-30T13:44:01Z | Bowen Sun, Chaozhuo Li, Yaodong Yang, Yiwei Wang et al.*

> TwinGate frames decompositional jailbreak defense as a stateful serving-time intent-tracking problem under anonymized, interleaved traffic.

This paper targets jailbreaks where a malicious objective is split across many individually benign-looking requests, which is hard to detect when production traffic is anonymized and interleaved. TwinGate uses a dual-encoder design with asymmetric contrastive learning to place intent-matched malicious fragments in a shared latent space, while a frozen parallel encoder is used to reduce false positives from benign topical overlap. The serving claim is that each request needs only one lightweight forward pass and can run alongside the target model's prefill phase, so monitoring overhead should be small. The abstract reports evaluation on a 3.62M-instruction dataset covering 8,600 malicious intents, with high recall, low false positives, robustness to adaptive attacks, and better latency/throughput than stateful and stateless baselines. The main missing details are the exact metrics, hardware and deployment setup, baseline implementations, and how negligible latency overhead is measured.

If the latency claim holds, this is useful for serving-layer safety systems that need cross-request intent tracking without adding a heavy monitor model to the critical path. A key caveat is that the abstract does not disclose exact latency, throughput, or false positive numbers. Another caveat is that the serving-overhead claim depends on hardware, concurrency, batching, and prefill scheduling details that are not specified in the abstract.

### 3. CuLifter: Lifting GPU Binaries to Typed IR

*arXiv preprint | 2026-04-30T06:34:54Z | Jisheng Zhao, Huanzhi Pu, Shinnung Jeong, Chihyo Ahn et al.*

> CuLifter targets the missing type layer in GPU binary lifting, recovering SASS register types so binaries can be translated into valid LLVM IR.

GPU binary analysis loses a key source of structure because compiler backends collapse typed values into an untyped register file. The paper frames type recovery as the central obstacle for lifting SASS into useful LLVM IR. CuLifter combines constraint-propagation-based register type recovery, conflict detection, explicit control-flow reconstruction, and aggregation of multi-instruction patterns. ### 4. RCW-CIM: A Digital CIM-based LLM Accelerator with Read-Compute/Write

*arXiv preprint | 2026-04-30T03:52:24Z | Yan-Cheng Guo, Tian-Sheuan Chang, Jian-Wei Su*

> RCW-CIM proposes a fabricated digital compute-in-memory LLM accelerator that targets weight-update latency with read-compute/write scheduling, nonlinear operator fusion, and a WS-OCS dataflow.

Digital CIM is positioned as a way to reduce LLM inference data movement while retaining precision, but the abstract argues that weight update latency becomes a major bottleneck when model weights exceed one CIM macro. RCW-CIM addresses this with a read-compute/write architecture, nonlinear operator fusion for dependency-induced latency, and a weight-stationary/output-column-stationary dataflow for prefill. The abstract reports latency, DRAM-access, and internal weight-update reductions on INT4-weight Llama2 settings, and states that the chip was fabricated in TSMC 22 nm CMOS at 100 MHz with dual DDR5-6400 memory. The likely value is a hardware design that attacks both decode and prefill bottlenecks through data placement and update scheduling rather than only raw compute throughput. The biggest missing detail is how baselines, model coverage, accuracy impact from approximation, and end-to-end serving assumptions were evaluated.

If the evaluation holds, the work is useful for understanding how CIM accelerator dataflow and update scheduling can affect real LLM prefill and decode latency. A key caveat is that baseline definitions and evaluation setup are not clear from the abstract. Another caveat is that approximation effects on LLM quality are not described in the abstract despite the claimed nonlinear operator fusion gains.

### 5. From Threads to Tiles: T2T, a Compiler for CUDA-to-NPU Translation via 2D Vectorization.

*CGO | 2026 | Shuaijiang Li, Jiacheng Zhao, Ying Liu 0055, Shuoming Zhang et al.*

> T2T translates CUDA kernels to tile-based NPU instructions by lifting SIMT thread parallelism into analyzable 2-D loop structure and vectorizing loop pairs onto NPU tiles.

The paper targets the semantic mismatch between CUDA's fine-grained scalar thread model and NPUs that expose coarse-grained 2-D tile instructions. Its main compiler move is to convert CUDA SIMT parallelism into a Unified Parallelism Abstraction, then select loop pairs for 2-D vectorization onto tile hardware. The abstract claims the implementation, built on Polygeist, handles correctness-sensitive CUDA features such as control flow and warp-level intrinsics through semantics-preserving optimizations. Reported evidence includes evaluation on representative NPU architectures, up to 73% of native CUDA performance on an A100 GPU, and up to 6.9x speedup over baseline translation approaches. The biggest missing details are the exact NPU targets, benchmark mix, baseline implementations, and how broadly the approach handles irregular CUDA kernels.

If the claims hold, the work offers a practical compiler path for reusing CUDA kernels on tile-oriented accelerators instead of rewriting kernels manually for each NPU backend. A key caveat is that the abstract does not disclose the exact hardware targets, making portability and hardware-dependence hard to judge. Another caveat is that the 73% comparison to native CUDA on A100 needs careful interpretation because the target is NPU execution but the reference is GPU performance.

### 6. Strait: Perceiving Priority and Interference in ML Inference Serving

*arXiv preprint | 2026-04-30T17:55:28Z | Haidong Zhao, Nikolaos Georgantas*

> Strait is a GPU inference serving system that predicts transfer contention and kernel interference to schedule dual-priority traffic with fewer high-priority deadline misses.

GPU inference serving becomes harder under high utilization because concurrent data movement and kernel execution can make latency estimates unreliable. Strait targets dual-priority inference traffic, especially on-premises settings where high-priority requests need stronger deadline satisfaction without fully sacrificing low-priority work. Its mechanism is an adaptive latency prediction model that accounts for data-transfer contention and kernel execution interference, then feeds those predictions into priority-aware scheduling. The abstract reports evaluations under intense workloads and claims high-priority deadline violations drop by 1.02 to 11.18 percentage points, with acceptable low-priority cost and more equitable behavior than software-defined preemption. The main missing detail is the evaluation surface: model types, GPU hardware, workload construction, baseline configuration, and whether the setting maps to LLM serving are not clear from the abstract.

If the results hold, the practical value is better priority isolation for busy GPU inference clusters without relying mainly on preemption. A key caveat is that the abstract does not show whether the workloads are LLM-like or limited to smaller DNN inference tasks. Another caveat is that hardware assumptions, deployment scale, and reproducibility details are not provided in the abstract.
