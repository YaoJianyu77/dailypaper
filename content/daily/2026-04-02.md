### 1. Q-Hitter: A Better Token Oracle for Efficient LLM Inference via Sparse-Quantized KV Cache.

*MLSys | 2024 | Zhenyu Zhang 0015, Shiwei Liu 0003, Runjin Chen, Bhavya Kailkhura et al.*

> The paper studies joint KV-cache sparsification and low-bit quantization, arguing that token selection should account not just for attention importance but also for whether selected tokens remain robust after quantization.

This paper targets a central LLM inference bottleneck: KV-cache memory footprint and bandwidth cost at long context lengths. It argues that simply combining heavy-hitter token sparsification with low-bit KV-cache quantization is not enough, because aggressive quantization can distort token selection and because the most attention-important tokens are not always the best ones to quantize. The proposed Q-Hitter rule combines accumulated attention scores with a layer-dependent notion of quantization friendliness to choose tokens that are both important and more tolerant to low-bit storage. The abstract supports the motivation and the failure modes of the naive combination, but it does not clearly state the full experimental setup, exact gains, or how quantization friendliness is computed in practice. If the method is sound, it is relevant for bandwidth-limited LLM serving, but the real value depends on fairness against prior KV-cache baselines and implementation cost.

If the selection rule holds up, it could make KV-cache compression more deployable by improving the tradeoff between memory savings, bandwidth reduction, and model quality under low-bit inference constraints. A key caveat is that the abstract is underspecified on experimental evidence, so the strength of the claimed gains is unclear. Another caveat is that a quantization-friendliness metric may introduce runtime or tuning overhead that weakens systems value in practice.

### 2. From Threads to Tiles: T2T, a Compiler for CUDA-to-NPU Translation via 2D Vectorization.

*CGO | 2026 | Shuaijiang Li, Jiacheng Zhao, Ying Liu 0055, Shuoming Zhang et al.*

> This paper presents a compiler that translates CUDA kernels to NPU-style 2-D tile programs by making SIMT parallelism explicit and then vectorizing selected loop pairs onto tile instructions.

This paper targets the growing mismatch between CUDA s fine-grained thread model and NPUs that execute coarse-grained 2-D tile instructions. It proposes T2T, a compiler pipeline that first rewrites implicit SIMT parallelism into an analyzable loop-based form through a Unified Parallelism Abstraction, then applies 2-D vectorization to map chosen loop pairs onto NPU tiles. The abstract claims both correctness-oriented handling of CUDA features such as control flow and warp-level intrinsics and performance gains over baseline translation methods, with translated kernels reaching up to 73% of native CUDA performance on an A100 GPU and up to 6.9x over baselines. For systems readers, the main value is the compiler/runtime co-design angle for porting CUDA kernels onto tile-centric accelerators rather than hand-rewriting kernels. The biggest missing detail from the abstract is how fair the baselines are and what hardware assumptions, benchmark mix, and translation limits drive the reported results.

If the method is robust, it could reduce manual porting effort for CUDA software stacks and provide a principled compiler path for retargeting kernels to tile-based accelerators. A key caveat is that the headline comparison mixes translated NPU-style execution with native CUDA performance on an A100, so the practical meaning of that ratio needs careful reading. Another caveat is that the abstract does not clarify baseline strength, benchmark composition, or whether results transfer to LLM-serving kernels rather than generic CUDA workloads only.

### 3. LLM-VeriOpt: Verification-Guided Reinforcement Learning for LLM-Based Compiler Optimization.

*CGO | 2026 | Xiangxin Fang, Jiaqin Kang, Rodrigo Rocha, Sam Ainsworth 0001 et al.*

> This paper trains a small LLM to perform LLVM IR peephole optimization by using formal verification signals as reinforcement-learning feedback, aiming to improve both optimization usefulness and semantic correctness.

This paper targets a core blocker for LLM-based compiler optimization: generated low-level rewrites are often incorrect, which makes them unsafe and hard to improve iteratively. The authors propose LLM-VeriOpt, a reinforcement-learning setup that uses Alive2 semantic-equivalence checks inside the reward signal to train Qwen-3B with GRPO for LLVM IR peephole optimization. The abstract claims large gains over prompt-only use of the base model, including much higher rates of successful code modification, 90% verifiably correct output, and performance comparable to LLVM instcombine while occasionally surpassing it. For systems readers, the interesting point is the compiler-runtime co-design between LLM optimization generation and formal verification feedback rather than ordinary code generation alone. The biggest missing detail from the abstract is evaluation scope and fairness: it does not make clear what benchmarks, hardware, optimization budget, or verifier failure modes were involved.

If the claims hold, this suggests a practical path to smaller verifier-guided LLM optimizers that can generate low-level rewrites with materially better safety and competitive optimization quality. A key caveat is that the abstract does not specify enough about benchmarks or workload diversity to judge generalization. Another caveat is that reported speedups may depend heavily on the chosen baseline, optimization budget, and measurement protocol; the abstract alone is not enough to assess fairness fully.

### 4. Oaken: Fast and Efficient LLM Serving with Online-Offline Hybrid KV Cache Quantization.

*ISCA | 2025 | Minsu Kim 0004, Seongmin Hong, Ryeowook Ko, Soongyu Choi et al.*

> This paper presents a hybrid online-offline KV cache quantization design for LLM serving that targets the memory-bandwidth and memory-capacity limits of batched attention.

The paper targets a core LLM serving bottleneck: batched attention is constrained by memory bandwidth, while long-context KV caches also strain memory capacity and system cost. It argues that prior KV-cache quantization schemes can preserve accuracy with low-bit storage, but their online outlier detection overhead can erase the practical serving benefit. Oaken proposes a hybrid design that sets outlier thresholds offline and uses them online to drive quantization scale selection, coupled with custom quantization/dequantization engines and memory-management hardware. The claimed value is a better accuracy-performance trade-off for KV cache compression that could reduce bandwidth pressure and capacity overhead in serving stacks. The biggest missing detail is evidence: the abstract is truncated and does not clearly state the evaluation setup, quantitative gains, hardware assumptions, or baseline fairness.

If the co-design works as claimed, it could make KV-cache compression more deployable in long-context LLM serving by easing both bandwidth and capacity pressure without paying large online quantization overheads. A key caveat is that the abstract is incomplete, so the actual performance and accuracy evidence cannot be verified from the provided text. Another caveat is that benefits may depend on custom hardware support rather than transferring cleanly to existing GPU serving stacks alone.

### 5. Proteus: Portable Runtime Optimization of GPU Kernel Execution with Just-in-Time Compilation.

*CGO | 2025 | Giorgis Georgakoudis, Konstantinos Parasyris, David Beckingsale*

> This paper presents a portable JIT path that specializes GPU kernels from LLVM IR at runtime, aiming to beat static compilation by using input and launch-time information on both AMD and NVIDIA GPUs.

This paper targets a familiar systems gap in GPU computing: ahead-of-time compilation cannot exploit concrete runtime inputs and launch parameters, so optimized kernels can still leave performance on the table. Proteus addresses this with a language-agnostic JIT pipeline that extracts LLVM IR, specializes kernels at runtime through lightweight annotations, and tries to keep compilation overhead low enough for end-to-end wins. The abstract claims evaluation across diverse programs on AMD and NVIDIA GPUs, with end-to-end speedups over AOT compilation and average gains over CUDA-specific Jitify, suggesting the payoff comes from both better specialization and lower JIT overhead. For readers interested in compiler/runtime co-design, the main value is the portability claim across vendors rather than another CUDA-only tuning stack. The biggest missing detail is where the gains actually come from in each workload, how much runtime compilation costs, and how fair the comparison setup is across baselines and hardware.

If the method holds up, it offers a practical way to add portable runtime specialization to GPU kernels and could matter for systems that need cross-vendor performance tuning without rewriting around vendor-specific JIT tools. A key caveat is that the abstract does not say which transformations are applied at JIT time, so the mechanism behind the speedups is still opaque. Another caveat is that evaluation details are underspecified, which makes baseline fairness and reproducibility hard to judge from the abstract alone alone ?.

This paper targets a longstanding systems problem: high-performance GEMM depends on architecture-specific micro-kernels that are expensive to build and maintain by hand. The authors propose a step-by-step method in Exo for generating these micro-kernels, with the hardware target described through a concise library-level instruction specification rather than fully manual intrinsic or assembly coding. The abstract claims the resulting kernels perform close to, and sometimes better than, manually developed micro-kernels, which suggests a compiler-plus-DSL path to more portable low-level code generation. If that claim holds, the value is not just speed but a cleaner retargeting workflow for new hardware. The main missing detail is that the abstract does not make the evaluation setup, target architectures, baselines, or fairness criteria clear.

For compiler and kernel-generation work, the main payoff is a more portable path to expert-level micro-kernel construction if Exo can preserve performance across targets. A key caveat is that the abstract does not specify the experimental setup, so the strength and generality of the performance claim are unclear. Another caveat is that this is about GEMM micro-kernels rather than end-to-end LLM inference serving, so the connection to current inference stacks is indirect unless the techniques transfer to attention or matmul backends clearly.
