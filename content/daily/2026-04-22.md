### 1. Q-Hitter: A Better Token Oracle for Efficient LLM Inference via Sparse-Quantized KV Cache.

*MLSys | 2024 | Zhenyu Zhang 0015, Shiwei Liu 0003, Runjin Chen, Bhavya Kailkhura et al.*

> The paper argues that KV-cache sparsification and low-bit quantization should be co-designed, and proposes a token selector that balances attention importance with quantization friendliness.

Long-context LLM inference is often bottlenecked by KV-cache memory capacity and bandwidth, which is why both token sparsification and low-bit KV quantization have become attractive compression strategies. This paper argues that combining them naively is unstable: low-bit quantization can distort heavy-hitter selection during generation, especially in deeper layers, and tokens chosen only by accumulated attention may not be good quantization targets. Q-Hitter addresses this by selecting tokens with both high attention importance and layer-dependent quantization friendliness, aiming to keep the tokens that matter while making sparse-quantized KV caches more robust. The provided abstract supports this problem diagnosis and the existence of the proposed selector, but it does not clearly state the evaluation setup, baselines, or concrete end-to-end gains. The practical value is a more realistic co-design view of KV-cache reduction for serving, and the biggest missing detail is whether the method improves actual throughput-quality tradeoffs under fair deployment conditions.

If the claim holds, the method could make KV-cache sparsification and quantization compose more effectively, improving memory- and bandwidth-limited LLM inference rather than optimizing each step in isolation. A key caveat is that the provided abstract is truncated, so the exact algorithm, experiments, and improvement claims are underspecified. Another caveat is that quantization-friendliness scoring may require extra online bookkeeping that offsets part of the serving gain, and the abstract does not clarify this cost.

### 2. From Threads to Tiles: T2T, a Compiler for CUDA-to-NPU Translation via 2D Vectorization.

*CGO | 2026 | Shuaijiang Li, Jiacheng Zhao, Ying Liu 0055, Shuoming Zhang et al.*

> A compiler pipeline that lifts CUDA SIMT kernels into explicit loop structure and remaps them onto 2-D NPU tile instructions for automatic CUDA-to-NPU translation.

This paper targets the mismatch between CUDA's fine-grained SIMT thread model and NPUs that expose coarse, explicit 2-D tile instructions. It proposes T2T, a compiler framework that first rewrites implicit CUDA parallelism into an analyzable loop nest through a Unified Parallelism Abstraction, then selects loop pairs for 2-D vectorization onto NPU tile operations. The abstract says the pipeline also includes semantics-preserving handling for control flow and warp-level intrinsics, which are common pain points in CUDA translation. Reported evidence is a benchmark study on representative NPU architectures, with translated kernels reaching up to 73% of native CUDA performance relative to an A100 reference and outperforming baseline translators by up to 6.9x. The main missing detail is how performance varies across kernels, memory patterns, and hardware targets, since the abstract does not clarify the evaluation setup or failure cases.

If the method is robust, it provides a practical compiler route to reuse existing CUDA kernels on tile-based NPUs and reduce manual porting effort across accelerators. A key caveat is that the abstract leaves hardware assumptions vague, so the claimed generality across NPUs may be narrower than the headline suggests. Another caveat is that the performance summary may hide large variation across benchmark types, especially for memory-bound or irregular kernels, and the abstract does not say otherwise explicitly clearly means clear actually?.

### 3. LLM-VeriOpt: Verification-Guided Reinforcement Learning for LLM-Based Compiler Optimization.

*CGO | 2026 | Xiangxin Fang, Jiaqin Kang, Rodrigo Rocha, Sam Ainsworth 0001 et al.*

> This paper trains a small LLM peephole optimizer with verifier-backed reinforcement learning so semantic correctness becomes part of the optimization objective instead of a post-hoc check.

LLM-based compiler optimization is attractive, but generated low-level rewrites are rarely useful unless semantic correctness can be enforced reliably. This paper proposes LLM-VeriOpt, which trains Qwen-3B with GRPO and uses Alive2 verification signals in the reward loop so the model can learn from correctness feedback while producing LLVM IR peephole optimizations. The abstract claims a 5.4x increase in successfully modified code over prompt-only Qwen-3B, about 90% verifiably correct outputs, and speedups over O0 that are comparable to LLVM's handwritten `instcombine`, with some cases exceeding it. If those claims hold, the practical value is a path to small verifier-aware models that can generate deployable IR rewrites instead of merely plausible ones. The biggest missing detail is evaluation clarity: the abstract does not spell out benchmark coverage, inference-time verification cost, or how fair the baseline comparisons are.

If reproducible, this suggests a practical route to smaller verifier-aware models for LLVM IR optimization that improve usefulness without dropping semantic safety. A key caveat is that the abstract does not explain benchmark composition, so out-of-distribution generalization is hard to judge. Another caveat is that the deployment workflow is underspecified, especially whether verification remains necessary at inference time and what that costs in practice.

### 4. Oaken: Fast and Efficient LLM Serving with Online-Offline Hybrid KV Cache Quantization.

*ISCA | 2025 | Minsu Kim 0004, Seongmin Hong, Ryeowook Ko, Soongyu Choi et al.*

> This paper proposes a hybrid KV-cache quantization scheme for LLM serving that moves outlier-threshold selection offline and pairs it with custom accelerator support.

The paper targets a central serving bottleneck: batched attention is constrained by KV-cache memory bandwidth and capacity, and longer contexts make that pressure worse. It argues that prior mixed-bit KV-cache quantization schemes lose much of their benefit because online outlier detection is too expensive at serving time. Oaken's main idea is an online-offline hybrid design that sets outlier thresholds offline, then uses those thresholds for online quantization-scale decisions, alongside custom quantization/dequantization engines and memory-management units. The abstract claims this co-design can preserve accuracy while improving performance, and it says the authors built an Oaken accelerator, but the abstract is truncated and does not provide the evaluation setup, baselines, or quantitative results. For systems readers, the value is the attempt to turn KV-cache quantization into a serving-friendly path by removing online detection overhead, though the actual trade-off and hardware assumptions still need checking.

If the method holds up, it could reduce KV-cache bandwidth and capacity pressure without paying the usual online outlier-detection cost, which is directly relevant to long-context LLM serving efficiency. A key caveat is that offline thresholds may be brittle if serving distributions or model behavior shift from the calibration setting. Another caveat is that the claimed performance gains may rely heavily on custom hardware, limiting transfer to commodity GPU stacks or existing serving runtimes. The abstract is truncated, so the evaluation setup, baseline fairness, and reproducibility are not yet clear.

### 5. Proteus: Portable Runtime Optimization of GPU Kernel Execution with Just-in-Time Compilation.

*CGO | 2025 | Giorgis Georgakoudis, Konstantinos Parasyris, David Beckingsale*

> A portable LLVM-IR-based JIT runtime that specializes GPU kernels from runtime inputs and launch settings, with reported cross-vendor speedups but unclear overhead and workload breakdown.

This paper targets a standard AOT limitation in GPU software: some optimization opportunities only become visible once runtime inputs and launch parameters are known. Proteus proposes a lightweight, annotation-driven JIT path that extracts language-agnostic LLVM IR and specializes GPU kernels at runtime, aiming to keep compilation overhead low while remaining portable across AMD and NVIDIA GPUs. The abstract reports evaluation on a diverse program set, with end-to-end speedups up to 2.8x on AMD and 1.78x on NVIDIA over AOT, plus an average 1.23x gain over CUDA-specific Jitify. For systems readers, the main interest is the runtime/compiler co-design angle and the cross-vendor portability rather than another CUDA-only kernel tuning path. The biggest missing detail is where the gains actually come from, because the abstract does not make the kernel mix, overhead amortization, or baseline fairness clear.

If the claims hold under closer inspection, this offers a practical path to portable runtime kernel optimization for GPU software stacks that need late specialization without committing to a CUDA-only workflow. A key caveat is that the abstract does not show whether the wins come from a few favorable kernels or generalize across kernel classes. Another caveat is that the JIT overhead story is underspecified, so the tradeoff may look different in short-running or latency-sensitive deployments like LLM serving.

### 6. Tackling the Matrix Multiplication Micro-Kernel Generation with Exo.

*CGO | 2024 | Adrián Castelló 0001, Julian Bellavita, Grace Dinh, Yuka Ikarashi et al.*

> This paper presents an Exo-based workflow for building GEMM micro-kernels that aims to approach or exceed hand-written intrinsic or assembly implementations while reducing per-architecture rewrite effort.

The paper targets GEMM micro-kernel generation, a long-standing bottleneck because high performance usually depends on architecture-specific low-level code. It proposes a step-by-step method in Exo for producing these micro-kernels from a concise instruction-level description of the target hardware rather than writing each kernel manually from scratch. The abstract claims the generated kernels perform close to, and sometimes better than, manually developed kernels written with intrinsics or assembly, while also improving portability. If that claim holds, the main value is a more maintainable path to retargeting dense linear algebra kernels across architectures. The main missing detail is that the abstract does not specify the hardware targets, benchmark setup, amount of manual tuning still required, or the strength and fairness of the comparisons.

For compiler and kernel engineers, the payoff is a possible route to lower-effort retargeting of dense linear algebra kernels without giving up most of the low-level performance normally associated with manual implementations. A key caveat is that the abstract does not specify whether the results hold broadly or only on a narrow set of architectures and kernels. Another caveat is that portability may still hide substantial manual expertise in the instruction descriptions and scheduling process, which the abstract does not quantify clearly.
