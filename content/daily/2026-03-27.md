### 1. Numerical Superoptimization for Library Learning

*arXiv preprint | 2026-03-25T20:47:15Z | Jonas Regehr, Mitch Briles, Zachary Tatlock, Pavel Panchekha*

> This paper reframes numerical library design as a search problem: mine candidate math primitives from superoptimization traces, rank them by counterfactual workload utility, and add only the primitives that materially improve downstream kernels.

This paper studies a more upstream question than kernel-level numerical optimization: not just how to optimize floating-point code given a math library, but which new primitives should exist in that library at all. It introduces numerical library learning and implements it in GrowLibm, which repurposes Herbie's search, equivalence handling, and cost modeling to mine, deduplicate, and rank candidate primitives against real workloads. The abstract reports that adding expert implementations of the learned primitives improves kernel speed by up to 2.2x at fixed accuracy, can raise achievable accuracy substantially in at least one case, and can also be recovered from optimized LLVM IR to deliver up to 5% end-to-end application gains. The practical value is a workflow for expanding numerical libraries in a workload-driven way instead of relying only on manual expert intuition. The main missing detail from the abstract is how robust the ranking, implementation effort, and benchmark fairness are across broader workloads and hardware/compiler settings.

If the method holds up, it offers a principled way to co-design numerical libraries and compiler recognition around workload value rather than only hand-chosen primitives. A key caveat is that the abstract gives strong headline gains, but the breadth of workloads is narrow and may not reflect general-purpose numerical software. Another caveat is that the usefulness of the learned primitives may depend on substantial expert implementation work that the abstract does not quantify clearly.
