### 1. Transactional Attention: Semantic Sponsorship for KV-Cache Retention

*arXiv preprint | 2026-04-13T10:51:46Z | Abhinaba Basu*

> This paper proposes an anchor-based KV-cache retention rule for dormant value tokens and claims it fixes an extreme-budget failure mode that standard compression methods miss.

This paper targets a specific weakness in KV-cache compression: semantically critical tokens such as credentials or configuration values can receive near-zero attention until they are suddenly needed, so standard eviction signals discard them. The proposed Transactional Attention mechanism uses structural anchor patterns like "key:" or "password:" to sponsor adjacent value tokens, and a TA-Fast variant removes attention dependence while remaining compatible with SDPA and FlashAttention. In the abstract, the method is evaluated at a very small cache budget of K=16 tokens and is reported to reach 100% credential retrieval where six named baselines get 0%, while also maintaining 100% accuracy across 200 function-calling trials. If those results hold beyond the described settings, the main value is a cheap, composable fix for a concrete reliability gap in compressed LLM inference. The biggest missing detail is scope: the abstract does not make clear how broad the workloads, prompt formats, hardware setup, and baseline tuning are, so the generality and fairness still need checking.

If the retention rule generalizes, it could provide a low-overhead way to close an important correctness and safety gap in KV-cache compression while remaining compatible with existing serving kernels and stacks. A key caveat is that the method may depend heavily on explicit textual anchor patterns and fail when important values are not locally marked. Another caveat is that the reported wins may be narrow to credential retrieval and similar function-calling templates rather than general long-context reasoning workloads.

### 2. Flow-Controlled Scheduling for LLM Inference with Provable Stability Guarantees

*arXiv preprint | 2026-04-13T05:03:16Z | Zhuolun Dong, Junyu Cao*

> This paper studies admission control for LLM inference and proposes a flow-control scheduler that limits how prompts enter the active set so systems with unknown decode lengths can remain stable under KV-cache growth.

This paper targets a core serving problem in LLM inference: decode lengths are unknown, so per-request KV-cache demand grows during generation and can push the system into overflow or unstable behavior. The proposed method is a simple flow-control framework that regulates the rate at which new prompts join the active set, framing scheduling as a stability problem rather than only a throughput optimization problem. The abstract claims both theory and practice: it derives a necessary condition for any stable system, gives sufficient conditions under which the proposed algorithm is provably stable, and reports experiments against common practical strategies. Those experiments are said to show better token throughput, request throughput, average and tail latency, and more stable KV-cache utilization. The main missing detail is how the controller is instantiated and evaluated in real serving setups, including workload assumptions, baselines, and the tradeoff between aggressiveness and admission delay.

If the guarantees and experiments hold under realistic workloads, the method could give serving systems a cleaner way to trade admission rate against KV-cache safety, tail latency, and sustained throughput. A key caveat is that the abstract does not specify the serving stack, hardware, or workload distribution, so external validity is unclear. Another caveat is that the theoretical model may simplify real systems enough that the guarantees do not transfer cleanly to production schedulers with multiple interacting policies.

### 3. Quantization Dominates Rank Reduction for KV-Cache Compression

*arXiv preprint | 2026-04-13T14:06:18Z | Samuel Salfati*

> This paper argues that, at fixed KV-cache storage budgets, low-bit quantization preserves transformer quality much better than rank reduction because it keeps the full attention geometry intact.

KV-cache growth is a central inference bottleneck, so this paper asks whether cache compression should discard dimensions or instead keep all dimensions and lower precision. It compares rank reduction, quantization, and hybrid baselines at matched storage budgets across five transformer models spanning 124M to 14B parameters, including both MHA and GQA settings. The claimed result is that quantization consistently beats rank reduction on perplexity, with especially sharp failures for matched-budget rank reduction on LAMBADA, while joint K+V INT4 quantization still gives large cache savings with only a small reported perplexity increase on Mistral 7B. The proposed reason is structural: removing dimensions can flip which token attention selects, whereas quantization noise is bounded and usually preserves score ordering, and the basis ablation is presented as evidence that this is not just a coordinate-choice artifact. The main missing detail is systems-side evidence: the abstract does not say how these methods compare in latency, bandwidth, kernel complexity, or deployment overhead on real hardware.

If the result generalizes, it strongly favors full-dimension low-bit KV storage over projection-based compression for practical LLM serving systems. A key caveat is that the abstract emphasizes quality outcomes but does not specify latency, throughput, memory-bandwidth, or implementation overhead. Another caveat is that the theoretical result may be locally persuasive, but the abstract does not show how tightly it predicts end-to-end behavior across settings beyond those reported here.

### 4. EdgeCIM: A Hardware-Software Co-Design for CIM-Based Acceleration of Small Language Models

*arXiv preprint | 2026-04-13T14:16:20Z | Jinane Bazzi, Mariam Rakka, Fadi Kurdahi, Mohammed E. Fouda et al.*

> A CIM-based edge accelerator co-designed with tile mapping to target memory-bound decode workloads in small language model inference.

This paper targets a real systems bottleneck in edge SLM serving: prefill can be handled reasonably well by GPUs, but autoregressive decode is dominated by memory-bound GEMV and wastes both bandwidth and energy. It proposes EdgeCIM, a hardware-software co-design built around a 65nm compute-in-memory macro plus a tile-based mapping strategy intended to balance pipeline stages and reduce DRAM pressure for end-to-end decoder-only inference. The abstract reports simulator-driven design-space exploration up to 4B-parameter models and claims substantial throughput and energy gains over NVIDIA Orin Nano and Qualcomm SA8255P, with additional averages across multiple SLM families under INT4. If these comparisons are methodologically fair, the practical value is a stronger edge inference path for real-time low-power decoding. The main missing detail is whether the reported gains come from a realistic end-to-end setup with comparable quantization, batching, memory capacity, and prefill/decode accounting rather than from favorable modeling assumptions.

If the setup is sound, the work could matter for edge LLM serving by showing that decode-centric hardware-software co-design can outperform general-purpose edge accelerators on tokens per second and tokens per joule. A key caveat is that the main results appear to rely on simulation, so implementation realism and system overheads need scrutiny. Another caveat is that baseline fairness is unclear from the abstract, especially around INT4, workload configuration, and end-to-end accounting conventions for competitor hardware.` `Hardware specificity may limit portability: a 65nm CIM design optimized for edge decode may not generalize across process nodes, models, or mixed workloads.

### 5. Record-Remix-Replay: Hierarchical GPU Kernel Optimization using Evolutionary Search

*arXiv preprint | 2026-04-13T07:25:12Z | Daniel Nichols, Konstantinos Parasyris, Caetano Melone, Tal Ben-Nun et al.*

> This paper presents a hierarchical GPU kernel optimization framework that combines LLM-guided evolutionary search, Bayesian optimization, and record-replay compilation to search source, compiler, and runtime choices together.

The paper targets a real GPU systems problem: performance tuning now spans source-level implementation decisions, compiler pass sequences and flags, and launch-time parameters, and that cross-layer space is too large for manual tuning. It proposes Record-Remix-Replay (R^3), a hierarchical framework that combines LLM-driven evolutionary search with Bayesian optimization and record-replay compilation so candidate variants can be evaluated quickly enough for end-to-end search. The abstract claims this lets the system optimize full scientific applications better than approaches that only tune kernel parameters or compiler flags, while being nearly an order of magnitude faster than modern evolutionary search methods. For systems readers, the main interest is the co-design across program representation, compiler search, and fast evaluation infrastructure rather than any single search heuristic. The biggest missing detail is that the abstract does not make the evaluation setup, hardware targets, benchmark coverage, or baseline fairness clear.

If the claims hold, this could make cross-layer GPU tuning substantially more practical and reduce the human effort needed to retarget performance-critical workloads to new architectures. A key caveat is that the abstract does not say whether the gains come from stronger search, faster evaluation, or both, which affects how reusable the idea is. Another caveat is that the workload emphasis is scientific applications, so relevance to LLM inference kernels is indirect until kernel types and hardware are shown clearly in the full paper.
