### 1. LookaheadKV: Fast and Accurate KV Cache Eviction by Glimpsing into the Future without Generation

*ICLR 2026 | 2026-03-11T15:44:32Z | Jinwoo Ahn, Ingyu Seong, Akhil Kedia, Junhan Kim et al.*

> This paper replaces draft-generation-based future-aware KV eviction with a learned importance predictor, aiming to keep eviction cheap while preserving long-context accuracy.

KV cache eviction matters because long-context autoregressive inference makes cache memory grow linearly, and prior "glimpse into the future" methods improve eviction quality but add expensive draft generation during prefilling. This paper proposes LookaheadKV, which augments transformer layers with parameter-efficient modules trained to predict true KV importance scores without explicitly generating surrogate responses. The abstract claims this keeps runtime overhead near inexpensive heuristics while outperforming stronger but costlier approximation baselines on long-context understanding benchmarks across multiple models. It also reports up to 14.5x lower eviction cost and faster time-to-first-token, making the work directly relevant to serving efficiency. The main missing details are how the modules are trained and integrated, what extra per-layer overhead they introduce, and how fair the hardware and baseline comparisons are.

If the reported tradeoff holds, this is a practical serving-side method for reducing KV-management overhead while keeping long-context quality higher than cheap heuristic eviction. A key caveat is that the abstract does not specify the training cost, training data, or whether the method must be retrained for each base model. Another caveat is that runtime gains may depend heavily on hidden assumptions about hardware, cache budget, batch size, or sequence-length regime.

### 2. AgentServe: Algorithm-System Co-Design for Efficient Agentic AI Serving on a Consumer-Grade GPU

*arXiv preprint | 2026-03-11T02:23:04Z | Yuning Zhang, Yan Yan, Nan Yang, Dong Yuan*

> This paper presents a single-GPU serving system for agentic LLM workloads that separates long and short phases of execution to reduce latency instability under overlapping requests.

The paper targets a serving problem that is becoming more important as small language models run locally as agents on consumer GPUs and interleave decoding with tool calls. It argues that agent workloads create a harder scheduling mix than standard chat because cold prefills, resume prefills, and short latency-critical decodes contend on one GPU and cause head-of-line blocking. AgentServe addresses this with prefill/decode isolation, dynamic budgeting for resume prefills, and GPU allocation through pre-established CUDA Green Context slots with adaptive control. The abstract claims improved latency stability with competitive throughput, including up to 2.8x better TTFT and 2.7x better TPOT than state-of-the-art baselines across multiple settings. The main missing detail is how these gains depend on the exact GPU, workload mix, baseline configuration, and the overheads of the proposed resource partitioning.

If the method holds up, it offers a practical serving design for local multi-agent execution on modest hardware where interactive latency matters more than maximizing raw batch throughput. A key caveat is that the abstract does not clarify whether the gains come from a broadly applicable scheduling idea or from workload-specific tuning. Another caveat is that cUDA Green Context dependence may limit portability or make the design sensitive to specific driver, runtime, or GPU assumptions that are not stated here.

### 3. Towards Cold-Start Drafting and Continual Refining: A Value-Driven Memory Approach with Application to NPU Kernel Synthesis

*arXiv preprint | 2026-03-11T14:57:06Z | Yujie Zheng, Zhuo Li, Shengtao Zhang, Hanjing Wang et al.*

> The paper presents EvoKernel, an agentic memory-based reinforcement learning framework that bootstraps and iteratively refines NPU kernels by retrieving prior experiences with stage-specific value estimates.

Kernel synthesis for emerging NPUs faces a data wall: general-purpose LLMs perform well in CUDA-rich settings but degrade sharply when domain data is scarce. This paper proposes EvoKernel, which treats drafting and refinement as a memory-based reinforcement learning problem and uses stage-specific Q-values to retrieve prior experiences that are most useful for either finding a feasible kernel or reducing latency. It also shares memory across tasks so knowledge from simpler operators can transfer to more complex ones. The abstract reports evaluation on an NPU variant of KernelBench, with correctness improving from 11.0% to 83.0% and a median 3.60x speedup over initial drafts after iterative refinement. The main missing detail is evaluation fidelity: the abstract does not explain the NPU stack, benchmark composition, baseline strength, or how the value estimates are learned and validated.

If the method is robust, it offers a practical way to adapt general-purpose LLMs to niche accelerator programming by turning prior synthesis attempts into reusable optimization knowledge. A key caveat is that the abstract does not specify the NPU hardware, compiler stack, or runtime conditions, so external validity is unclear. Another caveat is that the main comparison is against initial drafts, so baseline fairness versus stronger search or agentic alternatives is not established in the abstract alone.
