### 1. VitaLLM: A Versatile, Ultra-Compact Ternary LLM Accelerator with Dependency-Aware Scheduling

*arXiv preprint | 2026-04-30T04:07:21Z | Zi-Wei Lin, Tian-Sheuan Chang*

> VitaLLM is a hardware-software co-designed ternary LLM accelerator that targets edge inference bottlenecks with specialized ternary projection cores, mixed-precision attention support, KV-fetch pruning, and dependency-aware scheduling.

VitaLLM targets edge LLM inference, where ternary models reduce storage but still leave decode bandwidth, workload imbalance, and operation dependencies as bottlenecks on general-purpose hardware. The design combines TINT-Cores for ternary projections with a BoothFlex-Core for mixed-precision attention, then adds Leading One Prediction to reduce redundant KV-cache fetches and dependency-aware scheduling to hide nonlinear-operation latency. The abstract reports a TSMC 16nm implementation with 70.70 tokens/s decoding throughput, 0.223 mm^2 area, 65.97 mW power, and 17.4 TOPS/mm^2/W FOM, plus a bit-serial BoothFlex-BS extension for precision-agile inference. Its value is clearest for ultra-compact edge accelerators for ternary LLMs, especially if the reported decode bandwidth savings and utilization improvements hold under realistic model and sequence settings. The biggest missing detail is the evaluation context: model sizes, context lengths, baselines, workload mix, and whether the reported implementation is measured silicon or design-level evaluation are not clear from the abstract.

If the claims hold under fair workloads, the paper could be useful for accelerator designers studying ternary LLM inference, decode-stage bandwidth reduction, and compact edge hardware scheduling. A key caveat is that evaluation details are underspecified in the abstract, making the reported FOM and state-of-the-art comparison hard to judge. Another caveat is that the usefulness of Leading One Prediction depends on KV-cache access behavior that the abstract does not quantify beyond the claimed pruning role.

### 2. TwinGate: Stateful Defense against Decompositional Jailbreaks in Untraceable Traffic via Asymmetric Contrastive Learning

*arXiv preprint | 2026-04-30T13:44:01Z | Bowen Sun, Chaozhuo Li, Yaodong Yang, Yiwei Wang et al.*

> A stateful dual-encoder defense for decompositional jailbreaks that tracks malicious intent across anonymized, interleaved request streams with a claimed single-pass monitoring path during prefill.

Decompositional jailbreaks are a deployment-level LLM safety problem because malicious goals can be split across benign-looking requests while traffic remains anonymized and interleaved. TwinGate frames the defense as a stateful dual-encoder system: asymmetric contrastive learning clusters fragments that share malicious intent, and a frozen parallel encoder is used to reduce false positives from benign topical overlap. The abstract claims each request needs only one lightweight forward pass that can run alongside the target model prefill phase, so the serving argument is lower latency and better throughput than heavier generative monitors. Evidence is reported on a 3.62M-instruction dataset covering 8,600 malicious intents under a strictly causal protocol, with claims of high recall, low false positives, robustness to adaptive attacks, and better latency/throughput than stateful and stateless baselines. The biggest missing detail is the numeric evaluation setup: model sizes, hardware, traffic mix, baseline implementations, and exact overhead are not specified in the abstract.

If the claims hold, the method could add stateful jailbreak monitoring to LLM serving with lower online overhead than generative safety checks. A key caveat is that the abstract gives no numeric latency, throughput, or false positive values, so the systems claim cannot be judged from the summary alone. Another caveat is that the method may require nontrivial state management despite the anonymized traffic setting, but the abstract does not explain memory growth or eviction behavior.

### 3. CuLifter: Lifting GPU Binaries to Typed IR

*arXiv preprint | 2026-04-30T06:34:54Z | Jisheng Zhao, Huanzhi Pu, Shinnung Jeong, Chihyo Ahn et al.*

> CuLifter targets the type-erasure bottleneck in GPU binary lifting by recovering SASS register types and emitting typed LLVM IR.

GPU binary analysis is hard because compiler backends collapse source-level data types into a unified register file, leaving lifting tools without the type information needed for typed IR. This paper frames type recovery as the central obstacle and proposes CuLifter, a SASS-to-LLVM IR framework using constraint propagation with conflict detection, explicit control-flow reconstruction, and aggregation of multi-instruction patterns. The abstract reports evaluation across eight benchmark suites, 24,437 GPU functions, and 919 cubins from open-source applications, vendor libraries, and optimized ML runtimes, with 99.98% of functions lifted to valid LLVM IR. It also reports an ablation where disabling type recovery drops the x86 pass rate from 73.8% to 0%, supporting the claim that type recovery is critical. The missing detail is how semantic correctness, pass-rate measurement, and downstream usefulness are validated beyond producing valid IR.

If the lifting is semantically reliable, this could improve inspection, transformation, and analysis of optimized CUDA and ML-runtime binaries where source or compiler IR is unavailable. A key caveat is that valid LLVM IR does not necessarily prove semantic equivalence to the original GPU binary. Another caveat is that the abstract does not specify hardware generations, SASS coverage limits, or failure cases in detail ?.

### 4. RCW-CIM: A Digital CIM-based LLM Accelerator with Read-Compute/Write

*arXiv preprint | 2026-04-30T03:52:24Z | Yan-Cheng Guo, Tian-Sheuan Chang, Jian-Wei Su*

> The paper proposes a fabricated digital computing-in-memory accelerator for LLM inference that targets CIM weight-update latency with read-compute/write execution, nonlinear operator fusion, and a WS-OCS dataflow.

Digital CIM is positioned as a way to reduce DRAM-to-accelerator traffic for LLM inference while keeping digital precision, but the abstract argues that weight-update latency becomes a bottleneck when model weights exceed one CIM macro. RCW-CIM addresses this with a read-compute/write architecture, nonlinear operator fusion for dependency-induced latency, and a weight-stationary/output-column-stationary dataflow for prefill. The abstract reports 21.59% lower decoding compute latency on Llama2-7B, 69.17% lower latency from nonlinear operator fusion, and 49.76% overall prefill latency reduction for a 1024-token prefill, alongside reduced DRAM access and CIM weight updates. It also reports a fabricated TSMC 22 nm, 100 MHz chip reaching 3.28 TOPS, 42.3 TOPS/W, 4.2 ms prefill latency, and 26.87 decoded tokens/s for INT4-weight Llama2 with dual DDR5-6400 memory. The main missing detail is how baselines, accuracy impact from approximation, and workload generality are defined.

If the claims hold under fair baselines, the work is directly relevant to CIM-based LLM serving because it attacks both decode-time weight updates and prefill memory traffic. A key caveat is that baseline fairness is unclear from the abstract. Another caveat is that approximation-related accuracy impact is not described in the provided text.

### 5. Strait: Perceiving Priority and Interference in ML Inference Serving

*arXiv preprint | 2026-04-30T17:55:28Z | Haidong Zhao, Nikolaos Georgantas*

> Strait is a GPU inference serving system that predicts transfer contention and kernel interference to schedule dual-priority DNN traffic with fewer high-priority deadline misses.

GPU inference serving often runs mixed traffic under high utilization, where concurrent execution makes latency estimates and priority handling fragile. Strait targets dual-priority DNN inference by modeling data-transfer contention and kernel-execution interference, then using those predictions for priority-aware scheduling. The abstract claims evaluation under intense workloads reduces high-priority deadline violations by 1.02 to 11.18 percentage points while imposing acceptable costs on low-priority tasks. It also claims more equitable performance than software-defined preemption approaches. The main missing details are the model mix, GPU setup, workload construction, and baseline configuration, so the strength of the systems claim needs a full read.

If the evaluation is fair, the method could be useful for GPU serving stacks that need priority differentiation without relying primarily on preemption. A key caveat is that the abstract is not LLM-specific and only says DNN inference, so fit with LLM serving depends on the evaluated models. Another caveat is that hardware, workload, and baseline details are absent from the abstract, limiting confidence in generality and fairness claims.
