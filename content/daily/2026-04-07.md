### 1. TriAttention: Efficient Long Reasoning with Trigonometric KV Compression

*arXiv preprint | 2026-04-06T17:58:42Z | Weian Mao, Xi Lin, Wei Huang, Yuxin Xie et al.*

> This paper proposes a RoPE-aware KV cache compression method that scores key importance from pre-RoPE Q/K structure instead of recent post-RoPE attention, targeting long-reasoning memory bottlenecks.

This paper studies KV cache compression for long-reasoning LLM workloads, where memory growth and cache bandwidth become the main deployment bottlenecks. The key claim is that post-RoPE queries are a poor basis for importance estimation, so the method moves to pre-RoPE space and uses stable Q/K centers plus a trigonometric distance-preference model, along with Q/K norms, to rank keys for retention. The abstract reports that on AIME25 with 32K-token generation, the method matches full-attention reasoning accuracy while delivering either higher throughput or much lower KV memory use, and that it makes single-consumer-GPU deployment feasible for a long-context OpenClaw setup. The practical value is clear if the scoring rule is cheap and robust, because it would turn KV compression from a heuristic cache pruning trick into a more structure-aware long-context serving mechanism. The main missing details are baseline fairness, runtime overhead of the scoring logic, hardware setup, and how well the claimed Q/K concentration behavior generalizes beyond the reported setting.

If the method holds up, it could materially improve long-context LLM serving by reducing KV footprint enough to unlock cheaper hardware targets without giving up reasoning quality. A key caveat is that the claimed gains may depend heavily on the specific benchmark, context length, or model rather than general long-context serving workloads. Another caveat is that the abstract does not clarify the computational overhead of the scoring method, so throughput gains may be sensitive to implementation details and hardware balance between compute and memory bandwidth.

### 2. GENSERVE: Efficient Co-Serving of Heterogeneous Diffusion Model Workloads

*arXiv preprint | 2026-04-06T01:02:02Z | Fanjiang Ye, Zhangke Li, Xinrui Zhong, Ethan Ma et al.*

> This paper proposes a diffusion serving system that co-schedules text-to-image and text-to-video requests by exploiting step-level predictability and preemptibility in diffusion inference.

This paper targets a practical serving problem: text-to-image and text-to-video diffusion workloads increasingly share GPU clusters, but their compute profiles and latency needs differ enough that existing systems miss SLOs. The core idea is to treat diffusion inference as a sequence of predictable, discrete steps, then use that structure for finer-grained resource control during co-serving. GENSERVE combines video preemption at step boundaries, elastic sequence parallelism with dynamic batching, and an SLO-aware scheduler that jointly allocates resources across concurrent requests. The abstract claims up to a 44% improvement in SLO attainment rate over a strongest baseline across diverse settings, which suggests the gains come from better scheduling and adaptation rather than a new model architecture. The main missing details are hardware assumptions, workload mix, baseline fairness, and the overheads of preemption and dynamic reconfiguration.

If the method is robust, it could improve utilization and tail-latency control for mixed diffusion serving clusters without changing the underlying generation models. A key caveat is that the abstract emphasizes SLO attainment but does not clarify throughput, utilization, or cost tradeoffs. Another caveat is that it is unclear whether the reported gains depend on specific workload predictability or carefully tuned cluster settings that may limit generality after a full read.

### 3. Agentic Code Optimization via Compiler-LLM Cooperation

*arXiv preprint | 2026-04-05T19:44:02Z | Benjamin Mikek, Danylo Vashchilenko, Bryan Lu, Panpan Xu*

> This paper proposes a compiler-LLM optimization pipeline that inserts LLM agents at multiple abstraction levels and uses testing plus orchestration to trade off aggressive rewrites against correctness.

This paper targets a real gap in program optimization: standard compiler passes are reliable but can miss opportunities that require higher-level semantic reasoning, while LLM rewrites can sometimes find better code but often break correctness. The proposed system combines both by placing LLM optimization agents alongside compiler tools at multiple abstraction levels, with a guiding LLM coordinating the process and a separate LLM agent generating tests to check correctness and performance. The abstract claims that this cooperative setup beats conventional compiler optimization and single-level LLM baselines, with reported speedups up to 1.25x. The practical value is the attempt to make LLM-driven optimization less brittle by constraining it inside a compiler-oriented workflow rather than treating code rewriting as a free-form generation problem. The biggest missing detail is evaluation quality: the abstract does not make clear what programs were used, how correctness was validated beyond generated tests, or how much compute budget the multi-agent search requires.

If the method is robust, it could provide a more disciplined way to use LLMs for performance optimization by embedding them inside a compiler-and-testing loop rather than relying on unconstrained code rewrites. A key caveat is that the reported gain ceiling in the abstract is modest, so orchestration overhead may matter. Another caveat is that correctness assurance may be weak if it depends heavily on generated tests rather than stronger validation methods.
